# T-085: Шумоподавление (RNNoise WASM)

**Phase:** 5 — Quality of life
**Model:** Opus 4.7 · `high`
**Estimate:** ~3 часа

## Контекст
В голосе фоновый шум, клавиатура, кулер мешают. Решение — обработка микрофонного потока через RNNoise (open-source шумодав от Jean-Marc Valin/Mozilla, ~50KB WASM). Krisp — proprietary и платный, не наш случай.

## Файлы в скоупе
- `packages/polly/src/features/voice/noise-suppression.ts` (новый)
- `packages/polly/src/lib/livekit.ts` (доработка — pre-processing track)
- `packages/polly/src/features/settings/VoiceSettings.tsx` (toggle)

## Что сделать
1. **Используем готовую интеграцию**:
   - LiveKit имеет официальный пакет `@livekit/krisp-noise-filter` — НО это коммерческая Krisp, требует лицензии.
   - Альтернатива: `@livekit/track-processors` имеет RNNoise wrapper (бесплатный).
   - Или: подключить RNNoise.js напрямую через MediaStreamTrack processor API.
2. **Через `@livekit/track-processors`** (если доступно):
   ```ts
   import { BackgroundBlur, NoiseSuppressionProcessor } from '@livekit/track-processors'
   
   const processor = new NoiseSuppressionProcessor()
   await room.localParticipant.audioTracks.first().setProcessor(processor)
   ```
3. **Через MediaStreamTrack `processor`** (Chrome 94+):
   ```ts
   const stream = await navigator.mediaDevices.getUserMedia({ audio: true })
   const audioContext = new AudioContext()
   const source = audioContext.createMediaStreamSource(stream)
   const rnnoise = await loadRNNoiseModule()
   const processor = audioContext.createScriptProcessor(...)
   processor.onaudioprocess = (e) => rnnoise.process(e.inputBuffer, e.outputBuffer)
   // route processed stream to LiveKit
   ```
4. **UI toggle** в VoiceSettings: «шумоподавление (вкл/выкл)». Persist в localStorage.
5. **Performance check**: RNNoise ~5% CPU на одном core. На слабых машинах может быть заметно. Default — `false` для не-mac платформ, или auto-detect через `navigator.hardwareConcurrency < 4`.

## Out of scope
- AI-шумодав (Krisp tier) — нет.
- Echo cancellation вне браузерного — браузер сам ОК.

## Подсказки
- WebAssembly RNNoise: `@jitsi/rnnoise-wasm` пакет — рабочая версия, ~140 KB.
- AudioWorklet > ScriptProcessor (последний deprecated). Используй worklet, особенно для production-quality.
- Не пытайся обработать remote audio — это бессмысленно (это уже передано) и опасно (echo loops).
