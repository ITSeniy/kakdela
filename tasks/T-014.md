# T-014: Дефолтный сервер при francine seed

**Phase:** 1 — Текстовый дом
**Model:** Haiku 4.5 · `low`
**Estimate:** ~30-45 минут

## Контекст
Первый запуск инсталляции должен иметь дефолтный сервер и пару базовых каналов, иначе onboarding некуда деть пользователя. Это даже не «seed данных» в смысле фейковых, а скорее bootstrap инфраструктуры.

## Файлы в скоупе
- `packages/speedy/src/db/schema.ts` (таблицы `servers`, `channels`, `server_members`)
- `packages/francine/src/seed.ts` (новый)
- `packages/francine/src/index.ts` (вызов seed)

## Что сделать
1. **Таблицы** в schema.ts:
   - `servers`: id uuid pk, name text, iconUrl text nullable, ownerId uuid (nullable пока, fk поставим позже), createdAt
   - `channels`: id uuid pk, serverId fk, name text, kind enum('text','voice','dm'), category text nullable, topic text nullable, position int, createdAt
   - `server_members`: serverId fk, userId fk, role enum('owner','admin','member'), joinedAt. Primary key — composite (serverId, userId).
   - Enums через `pgEnum`.
2. **Миграция**: `pnpm --filter @kakdela/speedy db:generate`, проверить SQL, закоммитить.
3. **`seed.ts`**:
   - Проверить — есть ли хоть один server. Если есть — `console.log('already seeded')` и выйти.
   - Если нет: создать сервер `как у нас` (или с именем из env `KAKDELA_DEFAULT_SERVER_NAME`), 3 канала:
     - `#общее` (text, category «беседы», position 0)
     - `#флуд` (text, category «беседы», position 1)
     - `общая комната` (voice, без category, position 0)
   - Логировать ID созданного сервера — чтобы прокинуть в первый инвайт.
4. **CLI**: `pnpm francine seed` запускает seed. Идемпотентно.
5. Опционально (если просто): в конце seed вывести команду для создания первого инвайта: `pnpm francine invite create --server <id> --max-uses 1`.

## DoD
- [ ] Первый запуск seed создаёт server + 3 channels
- [ ] Повторный запуск ничего не делает, говорит «already seeded»
- [ ] В psql `SELECT * FROM channels` показывает 3 строки правильных типов
- [ ] `pnpm typecheck` чистый

## Out of scope
- Категории как отдельная таблица — пока строкой в channels, упрощаем.
- Кастомизация при seed (флаги CLI) — не нужно.

## Подсказки
- Не делай сервер с `ownerId = null` навсегда — это сразу даст проблемы с правами. В фазе 1 оставим null, в T-015 при первом регистрировавшемся пользователе сделаем его owner'ом дефолтного сервера через middleware.
