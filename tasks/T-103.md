## Что сделать

1. **Список секретных чатов** (`SecretChatList`):
   - Источник — локальная история (T-102), не сервер.
   - Строка: avatar + имя + превью последнего сообщения (из локального плейнтекста) + время + unread + **🔒-бейдж**.
   - Online-точка из presence (Redis/WS) — это **несекретные** метаданные, использовать можно.
   - Если показываем рядом cloud-DM — визуально отделить (замок vs без).
2. **Экран чата** (`SecretChatScreen`):
   - Шапка: `🔒 секретный · только на этом устройстве`, имя, presence.
   - Системный баннер сверху: «сообщения защищены сквозным шифрованием. история хранится только здесь».
   - `KD_DayDivider`, пузыри `SecretBubble`, composer (**пока только текст**).
   - **Без link-preview / без серверных эмбедов** (приватность + рендер локальный). Markdown-форматирование текста — можно, через существующий `markdown-it` + DOMPurify, но **без** запроса OG-превью.
3. **Пузырь** (`SecretBubble`):
   - Базируется на стиле `KD_DMBubble` (мои справа — `accent`/белый текст; чужие слева — `panel` + border).
   - Микро-иконка замка + статус `✓✓ прочитано` (по T-102 read-конверту).
4. **Старт секретного чата** (`StartSecretChat`):
   - Из профиля пользователя кнопка «начать секретный чат» → `crypto_session_exists?` → если нет: `GET /api/keys/:userId/bundle` (T-101) → установить сессию → открыть экран в состоянии «устанавливаем защищённое соединение…».
5. **Верификация** (`KeyVerification`):
   - Показать safety number (`crypto_safety_number`) сгруппированными mono-цифрами (стиль Signal: 60 цифр группами по 5).
   - Копия «сравните этот код с <имя> лично».
   - Кнопка «отметить проверенным» → флаг хранится **локально** (device-bound, не на сервере) → бейдж «проверено» в шапке.
6. **Баннеры сессии** (`SessionBanner`):
   - «ключ безопасности <имя> изменился — проверьте заново» при смене identity-ключа собеседника (libsignal untrusted identity) → **блокировать отправку** до подтверждения.
   - Device-bound онбординг-карточка при первом секретном чате: «история хранится только на этом телефоне. потеряете устройство — переписка исчезнет».
7. **Read-receipt / typing**: подключить к control-конвертам из T-102.

## Подсказки

- Presence/online — несекретные метаданные через существующий WS, использовать спокойно.
- Плейнтекст рендерить существующим пайплайном `markdown-it` + DOMPurify, но **выключить** fetch OG-превью для секретных.
- Формат safety number — как у Signal: 60 цифр, группы по 5, mono-шрифт (`--kd-mono`).
- Флаг «проверено» — **только локально**, на сервер не отправлять (device-bound).
- `SecretBubble` переиспользует геометрию `KD_DMBubble`, добавляя замочек; не копировать стили хардкодом — через токены.
- «Ключ изменился» — это сигнал безопасности, не косметика: блок отправки обязателен, иначе теряется смысл E2EE.
