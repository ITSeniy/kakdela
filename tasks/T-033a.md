# T-033a: Tauri — разрешение микрофона

**Phase:** 2 — Голос
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~1 час

## Контекст
WebView2 на Windows наследует разрешения от хоста (само Tauri-приложение). Если приложение не имеет разрешения — никакие browser-prompt'ы не помогут. Здесь — настройка capabilities Tauri 2 и понимание Windows permissions модели.

## Файлы в скоупе
- `packages/polly/src-tauri/capabilities/default.json` (расширить)
- `packages/polly/src-tauri/tauri.conf.json` (проверить)
- `packages/polly/src/features/voice/MicPermission.tsx` (новый — UI диалога если denied)
- `packages/polly/src-tauri/src/lib.rs` (опционально — команда проверки разрешения)

## Что сделать
1. **Capabilities** в `default.json` — Tauri 2 ACL:
   - Убедиться что у окна `main` есть permission `core:webview:allow-set-webview-focus`
   - Никаких специальных media permissions в Tauri 2 нет — getUserMedia работает «из коробки» в WebView2, но требует чтобы у самого приложения было системное разрешение.
2. **MicPermission.tsx** — компонент-диалог:
   - Показывается если `useVoiceRoom` бросил error 'no-mic-permission' или getUserMedia вернул `NotAllowedError`.
   - Текст: «Для голоса нужен доступ к микрофону. Откройте Параметры Windows → Конфиденциальность → Микрофон, включите «Разрешить настольным приложениям доступ к микрофону» и убедитесь, что «как дела» в списке.»
   - Кнопка «Открыть параметры» — через `@tauri-apps/plugin-shell` `open('ms-settings:privacy-microphone')`.
   - Кнопка «Попробовать снова» — повторный `getUserMedia` запрос, при успехе закрывает диалог.
3. **Linux**: на Linux нет аналога «системного permission gate» — webkit2gtk просто запросит у user'а при первом обращении. Для Linux MicPermission показывать другой текст с упоминанием PulseAudio/PipeWire.
4. **Detection платформы**: через `@tauri-apps/plugin-os` `platform()` — `'windows' | 'linux' | 'macos' | 'web'`. Web — это когда запущено через `pnpm dev:web` без Tauri.

## DoD
- [ ] При первой попытке голоса — нет краша, либо запрос на разрешение успешен, либо MicPermission диалог
- [ ] В Settings Windows → Микрофон отключен → диалог показывается с правильным текстом
- [ ] Кнопка «Открыть параметры» открывает правильную страницу Windows Settings (`ms-settings:privacy-microphone`)
- [ ] На Linux — диалог не показывает Windows-specific текст
- [ ] `pnpm typecheck` чистый

## Out of scope
- macOS — не наша целевая платформа.
- Camera permissions — не делаем камеру.

## Подсказки
- `ms-settings:privacy-microphone` работает только на Windows 10+. На старых — fallback на `control.exe /name Microsoft.PrivacySettings`. Но Win10+ — наш минимум.
- В Tauri 2 `shell.open` требует разрешения в capabilities: добавь `"shell:allow-open"` (с allow-list URL'ов).
- Не пытайся вызвать `permissions.query({name:'microphone'})` — он не работает в WebView2. Используй getUserMedia как probe.
