# T-050: setScreenShareEnabled + UI кнопка «демо»

**Phase:** 3 — Демонстрация экрана ⭐ MVP
**Model:** Opus 4.7 · `high`
**Estimate:** ~2 часа

## Контекст
Ключевая фича MVP. В WebView2 на Windows `getDisplayMedia` показывает системный picker (как в Chromium): выбор экрана / окна / вкладки приложения. Это нормально, не пугаемся.

## Файлы в скоупе
- `packages/polly/src/features/voice/store.ts` (расширить — screenSharing state)
- `packages/polly/src/features/voice/useScreenShare.ts` (новый)
- `packages/polly/src/features/voice/VoiceControls.tsx` (доработка)
- `packages/polly/src/lib/livekit.ts` (доработка — публикация screen track)

## Что сделать
1. **`useScreenShare()` хук**:
   - `startShare({ withAudio })` → `room.localParticipant.setScreenShareEnabled(true, { audio: withAudio })`. LiveKit сам вызовет `getDisplayMedia` и опубликует track.
   - `stopShare()` → `setScreenShareEnabled(false)`.
   - При успешном start → store.screenSharing = true; webhook отправит broadcast (T-032), который обновит state у других участников.
   - Catch ошибок:
     - `NotAllowedError` (отказ user'а в picker'е) → silent, screenSharing остаётся false.
     - `NotReadableError` (источник занят) → toast «не удалось захватить, попробуйте другой источник».
     - `OverconstrainedError` → fallback на дефолтные constraints (без resolution/frameRate).
2. **UI кнопка в VoiceControls** (теперь рабочая):
   - State idle: кнопка с иконкой экрана, при клике → `startShare({ withAudio: true })`.
   - State sharing (свой!): кнопка красная «остановить демо», + бейдж рядом «вы транслируете».
   - Disabled если уже кто-то другой шарит **и** LiveKit ограничивает (не наш случай — у нас N простreamов одновременно, см. T-054).
3. **Обработка остановки извне**:
   - User может остановить демо через нативный picker bar Chromium («Stop sharing» внизу экрана) — LiveKit поймает событие `LocalTrackUnpublished` для screen_share → обновить store.
4. **Self-preview**:
   - В `ParticipantTile` если это я и я шарю → ВНУТРИ tile'а показывать live preview моего экрана (через attach к моему local screen track). Это требует ParticipantTile (T-034) уметь показывать video.

## Подсказки
- LiveKit hide все низкоуровневые детали `getDisplayMedia` — НЕ вызывай его напрямую. Всё через `setScreenShareEnabled`.
- При первом screen share Windows может показать UAC-подобный диалог (для capture protection). Это нормально, user должен дать согласие один раз.
- В WebView2 dev mode — DevTools работают, но Chromium picker UI **внутри** WebView2 (а не отдельным окном). На прод-сборке (`tauri build`) picker — нативный системный chooser, в общем работает идентично.
