# T-021: Markdown в сообщениях

**Phase:** 1 — Текстовый дом
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~1-1.5 часа

## Контекст
Discord-подобное подмножество: `**bold**`, `*italic*`, `~~strike~~`, `__underline__`, `\`code\``, `\`\`\`fenced\`\`\``, `> quote`, `[link](url)`, упоминания `@user` и каналы `#channel`. Никакого HTML.

## Файлы в скоупе
- `packages/polly/src/features/chat/markdown.ts` (новый)
- `packages/polly/src/features/chat/Message.tsx` (использовать markdown.render)
- `packages/polly/src/features/chat/mentions.ts` (новый — парсер @user / #channel)

## Что сделать
1. **deps**: `markdown-it`, `dompurify`.
2. **`markdown.ts`**:
   - Инстанс markdown-it с конфигом:
     - `html: false`, `linkify: true`, `breaks: true` (newline без пустой строки → `<br>`)
     - Отключить плагины которые опасны (image — отключить, table — оставить)
     - Включить плагин для `__underline__` (или кастомный rule — см. подсказку)
   - Метод `render(text): string` — возвращает безопасный HTML после DOMPurify.
3. **`mentions.ts`**: pre-processing перед markdown:
   - `@<displayName>` → `<span data-mention="user" data-id="...">@DisplayName</span>` (через lookup в server-members кэше)
   - `#<channelName>` → `<a data-mention="channel" data-id="...">#name</a>`
   - Возвращает обработанную строку для передачи в markdown.
4. **`Message.tsx`** — заменить plain text на `dangerouslySetInnerHTML={{ __html: rendered }}`. DOMPurify санитизация делается **внутри** `markdown.render`, на верхнем уровне ничего не доверяем.
5. **Кликабельность**:
   - Делегирование в `MessageList`: `onClick` на корне, проверка `e.target.dataset.mention`. Клик по `user` → открыть профиль (T-068), по `channel` → переключить канал.
6. **Code blocks** — добавить класс для подсветки (через `highlight.js` или просто моно-шрифт + бордер; для MVP — без highlight'а).

## DoD

- [ ] `[link](https://example.com)` — кликабельная ссылка, в Tauri открывается в дефолтном браузере (через `@tauri-apps/plugin-shell` → `open`)

## Out of scope
- Подсветка синтаксиса в код-блоках — фаза 4 при желании.
- Emoji shortcodes (`:smile:`) — фаза 4-5.
- LaTeX — нет.

## Подсказки
- DOMPurify умеет конфигурироваться: `ADD_ATTR: ['data-mention', 'data-id']` чтобы не вычищало наши data-атрибуты.
- markdown-it по умолчанию не поддерживает `__underline__` (это в коммонмарке strong). Сделать через `core.ruler` или плагин `markdown-it-mark` с кастомным маркером.
- Кликом по ссылке открываем через Tauri shell.open, иначе она откроется внутри WebView2 (ломая UX).
