## Что сделать

1. **`api-types.ts`** — Zod-схемы и инферированные типы:
   - `UserSchema`: `id`, `username`, `displayName`, `avatarUrl`, `status` ('online'|'idle'|'dnd'|'offline'), `customStatus?`
   - `ServerSchema`: `id`, `name`, `iconUrl?`
   - `ChannelSchema`: `id`, `serverId`, `name`, `kind` ('text'|'voice'), `category?`, `topic?`, `position`
   - `MessageSchema`: `id`, `channelId`, `authorId`, `content`, `replyToId?`, `createdAt`, `editedAt?`
   - `ServerMemberSchema`: `serverId`, `userId`, `role` ('owner'|'admin'|'member'), `joinedAt`

   Шаблон:
   ```ts
   import { z } from 'zod'
   export const UserSchema = z.object({
     id: z.string().uuid(),
     username: z.string().min(2).max(32),
     displayName: z.string().max(64),
     avatarUrl: z.string().url().nullable(),
     status: z.enum(['online','idle','dnd','offline']),
     customStatus: z.string().max(128).nullable().optional(),
   })
   export type User = z.infer<typeof UserSchema>
   ```

2. **`ws-events.ts`** — discriminated union по полю `t`:
   ```ts
   export type ServerEvent =
     | { t: 'ready'; user: User; servers: Server[] }
     | { t: 'msg.new'; channelId: string; message: Message }
     | { t: 'msg.edit'; channelId: string; messageId: string; content: string; editedAt: string }
     | { t: 'msg.delete'; channelId: string; messageId: string }
     | { t: 'presence'; userId: string; status: User['status'] }
     | { t: 'typing'; channelId: string; userId: string }
     | { t: 'voice.join'; channelId: string; userId: string }
     | { t: 'voice.leave'; channelId: string; userId: string }
     | { t: 'voice.state'; channelId: string; userId: string; muted: boolean; screen: boolean }
     | { t: 'member.join'; member: ServerMember }
     | { t: 'member.leave'; userId: string }

   export type ClientEvent =
     | { t: 'hello'; token: string }
     | { t: 'typing'; channelId: string }
     | { t: 'presence'; status: 'online'|'idle'|'dnd' }
     | { t: 'ping' }
   ```

   Также экспортировать Zod-схему `ServerEventSchema` (через `z.discriminatedUnion('t', [...])`), чтобы клиент мог парсить пришедшие сообщения с валидацией.

3. **`design-tokens.ts`** — перенести `KD_LIGHT` и `KD_DARK` из `designs/final-chrome.jsx` в TS-объекты:
   ```ts
   export const KD_LIGHT = { bg: '#e8e0cc', /* ... */ } as const
   export const KD_DARK  = { bg: '#1a1610', /* ... */ } as const
   export type KdTheme = typeof KD_LIGHT
   ```
   Это для разных мест (например, иконка-аватарка делается на canvas с цветом из токенов).

4. **`index.ts`** — реэкспорт всего:
   ```ts
   export * from './api-types'
   export * from './ws-events'
   export * from './design-tokens'
   ```
