## Что сделать
1. **Context menu** на hover сообщения:
   - Reply, Edit (только для своих), Delete (для своих или если admin), Copy text, Copy link.
   - Edit/Delete показываются только если user — автор или (для delete) admin/owner сервера.
2. **Inline edit**:
   - Клик «edit» → message content заменяется textarea с тем же текстом + кнопками «save», «cancel».
   - Enter → save, Escape → cancel.
   - Save → PATCH /api/messages/:id, optimistic update.
   - При edit `editedAt` обновляется, в UI появляется `(изм.)` маркер мелкими буквами.
3. **Delete confirmation**:
   - Маленький confirm dialog (или просто Cmd+click для skip, как Discord).
   - DELETE /api/messages/:id, optimistic — сообщение исчезает.
4. **WS handlers** в useMessages:
   - `msg.edit` → найти сообщение в кэше, update content и editedAt.
   - `msg.delete` → убрать из кэша.
5. **Edge case: edit window expired**:
   - Backend ответит 403 `edit-window-expired`. UI: toast «не получилось редактировать — прошло больше 30 дней».
   - В UI можно дизейблить edit для сообщений старше 30 дней.

## Подсказки
- Context menu лучше делать через позиционируемый div (state: `{visible, x, y, messageId}`), а не native browser context menu — нативный нельзя стилизовать.
- Optimistic edit: сохрани старое значение перед PATCH, верни при ошибке. `useMutation` от TanStack делает это через `onError` callback.
