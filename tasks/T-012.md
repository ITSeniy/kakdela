# T-012: Экран Auth (login / register)

**Phase:** 1 — Текстовый дом
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~2-3 часа

## Контекст
Дизайн уже готов в `designs/final-auth.jsx`. Нужно переписать в нормальный React + TS, подключить к API из T-010.

## Файлы в скоупе
- `packages/polly/src/features/auth/AuthScreen.tsx` (новый)
- `packages/polly/src/features/auth/api.ts` (новый — функции login/register)
- `packages/polly/src/features/auth/store.ts` (новый — Zustand: текущий user, accessToken)
- `packages/polly/src/lib/api.ts` (новый — fetch wrapper с auth header и refresh)
- `packages/polly/src/lib/host/secrets.ts` (новый — абстракция над stronghold)

## Что сделать
1. **`lib/api.ts`**: `apiFetch<T>(path, opts)`:
   - подставляет `Authorization: Bearer <accessToken>` из store
   - на 401 пытается refresh (один раз), при успехе ретраит исходный запрос
   - кидает `ApiError` с `code`, `message`, `status`
2. **`lib/host/secrets.ts`**: интерфейс `{ get(key), set(key, value), delete(key) }`. Web-режим — sessionStorage, Tauri — stronghold (или fallback на appdata). Определить рантайм через `'__TAURI__' in window`.
3. **`features/auth/api.ts`**:
   - `register({ inviteCode, username, displayName, email, password })`
   - `login({ email, password })`
   - `logout()`
   - Все сохраняют `accessToken` в store + `refreshToken` через secrets.
4. **`features/auth/store.ts`** (Zustand):
   - `user: User | null`
   - `accessToken: string | null`
   - `status: 'idle'|'loading'|'authed'|'unauthed'`
   - Действия: `setSession(user, token)`, `clear()`, `init()` (вызывается при старте — пытается refresh).
5. **`AuthScreen.tsx`** — переписать `final-auth.jsx` с реальной формой:
   - Tab/switch между login и register
   - Поля как в дизайне (email, password, displayName, inviteCode, etc.)
   - Стилизация через Tailwind + KD-токены (никаких inline-стилей из jsx)
   - Сабмит через `onClick` на кнопке (никаких `<form>` — см. CONVENTIONS)
   - Loading state с дизейблом кнопки и спинером
   - Ошибки от API — toast или inline под полем
6. **App.tsx**: при `status === 'unauthed'` показывает `<AuthScreen>`, при `'authed'` — пока заглушку «main app placeholder» (полноценный шелл в T-018).

## Подсказки
- `final-auth.jsx` использует local state как заглушку. Не переноси этот state — он только для рендера прототипа.
- В дизайне может быть кнопка «вход через Google/Discord» — игнорировать (мы self-host).
- На Windows при первом обращении к stronghold плагин попросит мастер-пароль. В T-012 можно использовать фиксированный пароль из env, в T-068 (профиль) дать выбор.
