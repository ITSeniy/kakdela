# T-037: Корректный teardown WebRTC

**Phase:** 2 — Голос
**Model:** Opus 4.7 · `high` (тонкие race conditions)
**Estimate:** ~2 часа

## Контекст
Самый частый источник багов в WebRTC — недочищенные соединения. Симптомы: дублирующиеся участники, утечки памяти, фантомные audio elements, ошибки «cannot use after disconnect». Задача — пройтись по всем местам, где можно выйти из голосового канала, и убедиться что cleanup полный.

## Файлы в скоупе
- `packages/polly/src/features/voice/useVoiceRoom.ts` (аудит)
- `packages/polly/src/lib/livekit.ts` (аудит)
- `packages/polly/src/app/App.tsx` (window beforeunload)

## Что сделать
1. **Сценарии teardown**, на каждый — проверка:
   - Клик «выйти» → `leaveChannel()`.
   - Переход на другой канал (text или другой voice) → `leaveChannel` в effect cleanup.
   - Закрытие окна Tauri → `window.addEventListener('beforeunload', ...)` синхронно закрывает room.
   - Logout → `leaveChannel` перед очисткой auth store.
   - Network disconnect длительный (>30s) → LiveKit фейлит, ставим `status='failed'`, очищаем participants.
2. **Что должно быть очищено**:
   - LiveKit Room: `room.disconnect()`.
   - Все `<audio>` elements: `track.detach()` снимает их с DOM. Если делали вручную через `appendChild` — `removeChild`.
   - Local microphone track: при disconnect LiveKit его сам остановит, но safe — `room.localParticipant.audioTracks.forEach(p => p.track?.stop())`.
   - Store: `participants` → empty Map, `activeSpeakers` → empty Set, `screenSharing → false`.
   - Pending timeouts (например, debounce из T-036) → clearTimeout.
3. **Тест воспроизведения багов**:
   - Быстрое переключение между voice-каналами (10 раз за 5 секунд) → проверить через `livekit-cli` что в LiveKit нет призраков.
   - F5 / Ctrl+R (в dev) → проверить что Room закрылся корректно.
   - В Chrome DevTools → Memory snapshot до и после серии join/leave → не должно быть утечек по `MediaStreamTrack`.
4. **`beforeunload` handler**:
   ```ts
   window.addEventListener('beforeunload', () => {
     const room = getRoom()
     if (room?.state === 'connected') room.disconnect()
   })
   ```
   Это последняя линия — Tauri может закрыть окно жёстко, но в большинстве случаев успеет.

## Подсказки
- LiveKit при ungraceful disconnect (закрытие WebSocket без disconnect()) понимает это и убирает участника через ~30s timeout. Это норма, но в идеале — disconnect синхронно при beforeunload.
- Tauri имеет событие `close-requested` на окно — можно подписаться и сделать асинхронный cleanup перед закрытием. Это лучше чем `beforeunload`.
- Не пытайся в `useEffect` cleanup делать асинхронные операции с await — React это не дождётся. Использовать fire-and-forget с then-catch.
