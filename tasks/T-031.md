# T-031: POST /api/voice/:channelId/join

**Phase:** 2 — Голос
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~1 час

## Контекст
Клиент дёргает этот эндпоинт перед подключением к LiveKit. Сервер проверяет права, выдаёт токен из T-030.

## Файлы в скоупе
- `packages/speedy/src/routes/voice.ts` (новый)
- `packages/ginzu/src/api-types.ts` (схема VoiceJoinResponse)

## Что сделать
1. **`POST /api/voice/:channelId/join`** (требует auth):
   - Проверка: канал существует и kind='voice'. Иначе 400 `not-a-voice-channel`.
   - Проверка: user — член сервера канала. Иначе 403.
   - Вызов `guido.issueToken({ userId, channelId, displayName: user.displayName })`.
   - Запись в Redis `voice:channel:<channelId>:users` (set) для трекинга кто в комнате (резервный механизм, основной — webhook из T-032).
   - Ответ: `{ token, url, room }` + `{ participants: [{ userId, displayName, isScreenSharing }] }` (snapshot — кто уже там, через `guido.listParticipants`).
2. **`POST /api/voice/:channelId/leave`** (опционально, для явного выхода):
   - Просто удаляет из Redis (LiveKit сам почистит при disconnect).
   - Это не блокирует MVP — клиент может просто отключиться от LiveKit без вызова leave.
3. **`GET /api/voice/:channelId/participants`** — список текущих участников голоса (для UI в text-каналах рядом с voice-каналом — «кто в комнате»). Кэшируем 5 секунд в Redis.

## DoD
- [ ] POST /api/voice/:id/join для voice-канала с членством → 200 с токеном
- [ ] Тот же эндпоинт для text-канала → 400 `not-a-voice-channel`
- [ ] Тот же эндпоинт от не-члена → 403
- [ ] При 5 одновременных join'ах от разных users → у всех в `participants` snapshot'е они все видны
- [ ] `pnpm typecheck` чистый

## Out of scope
- Лимит участников на канал — LiveKit имеет `max_participants` в room config, на 15-20 человек ставим 25 и забываем.
- Bandwidth-управление — фаза 3.

## Подсказки
- Не пытайся переиспользовать токен (`if user уже в комнате → выдать тот же токен`) — LiveKit JWT stateless. Каждый join → новый токен с новым TTL.
- Snapshot участников может быть пустой (если комната ещё не создана в LiveKit) — `listParticipants` вернёт `[]`, это норма.
