**Phase:** 4 — Уют

## Что сделать
1. **Эндпоинты**:
   - `GET /api/users/:id` → public profile: `id, username, displayName, avatarUrl, customStatus, status, joinedAt` (но только если есть общий сервер). Если нет — 404.
   - `PATCH /api/me` → обновление своего профиля: `displayName, customStatus, avatarUrl, password (with currentPassword)`.
   - `POST /api/me/avatar` → upload (через тот же file flow из T-063).
2. **`ProfileModal.tsx`** — общая модалка для любого user'а:
   - Header: avatar большой, displayName, username, presence dot.
   - Body:
     - Custom status (если есть).
     - Список общих серверов с ролями.
     - Дата join'а.
     - Если это **не я** — кнопки «написать сообщение» (→ T-064 DM) и «упомянуть» (→ insert @ в composer).
     - Если **я** — кнопка «редактировать профиль» → переключение в edit mode.
3. **`ProfileEditForm.tsx`** — для своего профиля:
   - Avatar upload (drag-n-drop preview, crop до квадрата на client side).
   - displayName input.
   - customStatus input (max 128 chars).
   - Theme picker (light/dark/system) — реюз из T-022.
   - Voice settings — реюз из T-035.
   - Change password (требует currentPassword).
4. **Permissions**:
   - Username и email — нельзя менять (это identity).
   - Avatar — JPEG/PNG/WebP, max 2 MB, обязательный crop до 256×256 (через client-side canvas).

## DoD
- [ ] Клик по avatar в чате → открывается профиль автора
- [ ] Свой профиль — редактирование работает, изменения видны другим через WS presence event
- [ ] Avatar upload → сохранение → у всех обновляется через invalidation запроса /members
- [ ] Change password требует текущий пароль, при правильном — refresh-токен ротируется
- [ ] `pnpm typecheck` чистый

## Out of scope
- Богатый profile (bio, ссылки на соцсети, etc.) — нет.
- Friends list — у нас all-friends-by-default (member же).

## Подсказки
- Avatar crop на client: `<canvas>` + drawImage с правильными xy + toBlob. Никаких heavy libs нужно — react-easy-crop работает но 30KB. Самописное за 50 строк.
- Avatar URLs должны быть immutable (хэш в имени), чтобы можно было кэшировать вечно. Иначе после смены аватара у всех будет старый закэширован.
