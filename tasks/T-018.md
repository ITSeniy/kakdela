# T-018: Главный шелл — рельса серверов + каналы + члены

**Phase:** 1 — Текстовый дом
**Model:** Sonnet 4.6 · `medium` (Opus, если упрётся в сложный layout)
**Estimate:** ~3-4 часа (большая задача)

## Контекст
Это «коробка» приложения. Дизайн в `designs/final-chrome.jsx` — там же есть всё, что нужно (ServerRail, ChannelList, MemberList).

- `packages/polly/src/features/channels/ChannelList.tsx` (новый)

## Что сделать
1. **Установить deps**: `@tanstack/react-query`, `react-router-dom` (или wouter для лёгкости — wouter ~1KB).
2. **`query.ts`** — настроенный QueryClient (staleTime 60s для членов, 30s для серверов, без retry для 401/403/404).
3. **Компоненты-кирпичи** из `common.jsx`:
   - `<Avatar src size status>` — кружок с фолбэком на инициалы из displayName
   - `<ServerIcon iconUrl name active hasUnread>` — квадрат-«ромб» как в дизайне
   - `<StatusDot status>` — точка с цветом из токенов
4. **`ServerRail.tsx`** (левая узкая колонка):
   - Загрузка через `useQuery(['servers'], api.listServers)`
   - Кнопка «+» внизу — пока заглушка, в T-083 откроет модалку создания
   - Кнопка «домой» (для DM) — пока неактивна, в T-064
5. **`ChannelList.tsx`** (вторая колонка):
   - Заголовок сервера сверху с шевроном (открывает меню)
   - Под ним список каналов, сгруппированный по `category`. Категории сворачиваются.
   - Text-каналы и voice-каналы (последние с иконкой динамика, в T-034 добавим состояние «кто в комнате»)
   - Активный канал — подсвечен через `bg-kd-panel-hi`
   - Внизу — карточка текущего user'а с микрофон-mute / deafen / шестерёнкой (заглушки)
6. **`MemberList.tsx`** (правая колонка):
   - Список через `useQuery(['members', serverId], api.listMembers)`
   - Группировка по role + по статусу (онлайн сверху)
   - `<Avatar>` + displayName + customStatus + StatusDot
   - Клик по члену → откроет профиль (T-068)
7. **`Shell.tsx`** — layout:
   - `grid-cols-[64px_240px_1fr_240px]` (рельса серверов, каналы, чат, члены) на десктопе
   - Member list можно тогглить (для скриншаринга в фазе 3 он скрыт)
   - Внутрь чат-колонки роутер вставит ChatScreen или VoiceScreen (T-019, T-034)
8. **Роутинг** через wouter:
   - `/` → редирект на первый сервер, первый text-канал
   - `/servers/:serverId/channels/:channelId` — основной паттерн
   - `/dm/:userId` — DM (T-064)
9. **Восстановление последнего открытого канала** через localStorage `kd:last-channel`.

## Подсказки
- `final-chrome.jsx` — используется как **визуальный** референс, но state-флоу там фейковый. Переписывай в нормальный React с React Query.
- Не клади server list / channel list в один общий store — TanStack Query сама нормально кэширует. Zustand нужен только для UI state (открытый канал, member list visible, etc.).
- wouter API очень близок к react-router но в 10× меньше. Если есть опыт с react-router — можно его, но не тащи v6 и v7 в один проект.
