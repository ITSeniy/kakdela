# T-007: Caddy reverse-proxy конфиг для dev (опционально)

**Phase:** 0 — Фундамент
**Model:** Haiku 4.5 · `low`
**Estimate:** ~20 минут

## Контекст
В скаффолде уже есть `ops/caddy/Caddyfile.dev` (шаблон). Цель — довести до состояния, когда `caddy run --config ops/caddy/Caddyfile.dev` проксит всё на правильные локальные порты. Это **опционально**: dev-режим Tauri ходит напрямую на `:3001` без Caddy. Caddy нужен, чтобы тестировать сценарий, близкий к проду.

## Файлы в скоупе
- `ops/caddy/Caddyfile.dev` (доработать)
- `README.md` (раздел «опциональный Caddy»)

## Что сделать
1. Проверить, что Caddy установлен (`caddy version`). Если нет — добавить в README ссылку на установку.
2. В `Caddyfile.dev` поднять `:8080` (HTTP, без TLS на локалке) с маршрутами:
   - `/api/*` → speedy:3001
   - `/ws` → speedy:3001 (с поддержкой WebSocket upgrade — Caddy делает это автоматически через `reverse_proxy`)
   - `/livekit/*` → livekit:7880 (`uri strip_prefix /livekit`)
3. Проверить вручную: `curl http://localhost:8080/api/healthz` отдаёт тот же ответ, что `curl http://localhost:3001/healthz`.

## DoD
- [ ] `caddy run --config ops/caddy/Caddyfile.dev` стартует без ошибок
- [ ] Все три маршрута работают
- [ ] В README — секция «опциональный Caddy для dev» с командой запуска

## Out of scope
- Прод-конфиг с автоматическим TLS — фаза 5.
- Self-signed сертификаты — пока не нужны.
