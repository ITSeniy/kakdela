# T-036: Active speaker highlight

**Phase:** 2 — Голос
**Model:** Haiku 4.5 · `low`
**Estimate:** ~30 минут

## Контекст
Это уже есть в T-033 (subscribe на `ActiveSpeakersChanged`) и T-034 (UI кольцо). Эта задача — отполировка: правильный дебаунс, плавная анимация кольца, не моргать при коротких звуках.

## Файлы в скоупе
- `packages/polly/src/features/voice/ParticipantTile.tsx` (доработка)
- `packages/polly/src/features/voice/store.ts` (доработка — дебаунс)

## Что сделать
1. **Дебаунс на «выключение» activeSpeaker**:
   - LiveKit убирает speaker из списка слишком резко (как только тихо стало). Это даёт некрасивые моргания при паузах в речи.
   - В store: при `setActiveSpeakers(newSet)` — добавления применяем сразу, удаления — с задержкой 500ms (`setTimeout`, отменяемый при повторном добавлении).
2. **Плавная анимация кольца в CSS**: `transition: box-shadow 200ms ease-out, transform 200ms ease-out`. На active: `box-shadow: 0 0 0 3px var(--kd-accent)`. На non-active — никакой тени.
3. **Volume meter** (опционально, как «бонус»):
   - LiveKit Track имеет `setupAudioAnalyser` для измерения уровня громкости.
   - Используем для маленькой полосы внизу tile'а (3-5 столбиков как equalizer).
   - Если сложно — отложить в фазу 5.

## DoD
- [ ] Кто-то говорит непрерывно → кольцо стабильно горит
- [ ] Кто-то делает паузу 200ms → кольцо НЕ моргает
- [ ] Появление/исчезновение кольца плавное, без дёргания
- [ ] `pnpm typecheck` чистый

## Out of scope
- Polished volume meter — фаза 5.

## Подсказки
- `clearTimeout` — обязательно перед `setTimeout` для того же user'а, иначе будут утечки таймеров.
- Не вешай animation через JS — только CSS transitions. JS animation на 20+ tile'ах одновременно — лагает.
