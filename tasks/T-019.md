## Что сделать
1. **`api.ts`**:
   - `listMessages(channelId, { before?, limit? })` → `{ messages, nextCursor }`
   - `sendMessage(channelId, { content, replyToId?, clientNonce? })`
   - `editMessage(id, content)`, `deleteMessage(id)`
2. **`useMessages.ts`**:
   - `useInfiniteQuery` с `queryKey: ['messages', channelId]`
   - `getNextPageParam: lastPage => lastPage.nextCursor`
   - При получении WS `msg.new` для этого channelId — `queryClient.setQueryData(...)` (append к первой странице) — НЕ invalidate (не дёргать сеть).
   - При `msg.edit` / `msg.delete` — точечный update в кэше.
3. **`Message.tsx`**:
   - Принимает Message + предыдущее сообщение → решает, нужна ли «шапка» (аватар + имя + время) или это «продолжение» того же автора в той же минуте.
   - Render content через `markdown-it` + DOMPurify (см. T-021 — пока plain text OK).
   - На hover — действия (реплай / реакция / опции) — заглушки кроме «копировать текст».
   - Если `editedAt` — маленький `(изм.)` после контента.
   - Если в кэше есть `replyToId` — мини-блок цитаты сверху.
4. **`MessageList.tsx`**:
   - Виртуализация **не нужна** на старте (50 сообщений / страница, 5 страниц = 250 — браузер справится). Если в фазе 5 окажется тормозно — `react-virtuoso`.
   - Auto-scroll вниз при появлении нового сообщения, **только если user был внизу**. Если он проскроллил вверх — не дёргаем. (Tracking через IntersectionObserver на «pseudo-bottom»-маркере.)
   - Подгрузка вверх при достижении верха через `fetchNextPage()`.
   - Маркер «непрочитанное с момента» — горизонтальная линия с надписью (как в дизайне). Логика: `lastReadAt` хранится в localStorage `kd:read:<channelId>`, обновляется при scroll-to-bottom.
5. **`Composer.tsx`**:
   - Textarea с auto-grow (Shift+Enter — newline, Enter — send).
   - Кнопки слева (вложение, эмодзи, gif) — заглушки.
   - При отправке: оптимистичное обновление (сразу показать сообщение с `status: 'sending'`) + `clientNonce` (uuid). При успехе — заменить на серверную версию. При ошибке — пометить красным и кнопкой «retry».
   - Очистка после успешной отправки.
   - При reply-mode — над composer'ом баннер «отвечаете на: <автор>: <preview>», крестик отменяет.
6. **`ChatScreen.tsx`**:
   - Header с именем канала, topic'ом, кнопками (поиск / закрепы / mute — заглушки)
   - `<MessageList>`
   - `<Composer>`

## Подсказки
- `final-chat.jsx` группирует сообщения умно — изучи логику, она там полезная.
- Не вызывай `setQueryData` внутри useEffect — это вызовет лишний рендер. Лучше через `queryClient.setQueryData` напрямую в обработчике WS события.
- Composer auto-grow: `textarea.style.height = 'auto'; textarea.style.height = textarea.scrollHeight + 'px'` после каждого input.
- Markdown оставь plain text для T-019, разметку добавим в T-021 — не блокируйся.
