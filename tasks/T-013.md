# T-013: Экран Onboarding (ввод инвайт-кода)

**Phase:** 1 — Текстовый дом
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~1-2 часа

> **Примечание (2026-05-24):** `designs/final-onboarding.jsx` — это **post-auth** welcome-экран с тремя картами (`зайти к друзьям` / `завести свою комнату` / `вернуться к своим`), который требует мульти-сервер UI. Этот сценарий переехал в **T-083** (фаза 5). T-013 закрыт только в части pre-auth invite-flow, описанной ниже; визуальный референс для него — поле ввода кода + превью сервера, отдельного дизайн-файла нет.

## Контекст
Это вступительный экран — user вводит инвайт, видит превью сервера (имя, иконку), потом переходит к регистрации.

## Файлы в скоупе
- `packages/polly/src/features/auth/OnboardingScreen.tsx` (новый)
- `packages/polly/src/features/auth/api.ts` (расширить — `lookupInvite(code)`)
- `packages/polly/src/app/Router.tsx` (новый — связка onboarding ↔ auth)

## Что сделать
1. **`api.ts`**: `lookupInvite(code)` — без auth, GET `/api/invites/:code`, возвращает `{ serverName, serverIcon, expiresAt }`.
2. **`OnboardingScreen.tsx`**:
   - Шаг 1: ввод кода. Большое поле в моно-шрифте, маска `xxxx-xxxx` (но отправляем без дефиса). Кнопка «продолжить» дизейблится пока < 8 символов.
   - Шаг 2: превью сервера — карточка с иконкой, именем, текстом «вас пригласили в *X*. зарегистрироваться?». Кнопка «вступить».
   - Шаг 3: переход на `<AuthScreen mode="register" inviteCode={code} />`.
   - Ошибки: «инвалид/истёк/исчерпан» — соответствующие тексты.
3. **`Router.tsx`** — простейший роутер на состояниях:
   - `auth.status === 'unauthed'` + URL вида `?invite=XXX` → onboarding
   - `auth.status === 'unauthed'` без инвайта → auth (с переключателем регистрация/вход)
   - `auth.status === 'authed'` → main app placeholder
   - `auth.status === 'loading'` → splash экран

## Out of scope
- Deep-links (`kakdela://invite/...`) — фаза 5.
- QR-код инвайта — не нужен на десктопе.

## Подсказки
- Маска `xxxx-xxxx` — `value.replace(/[^a-z0-9]/gi,'').toUpperCase().slice(0,8)`, потом вставляешь дефис при показе через `.slice(0,4) + '-' + .slice(4)`. Хранишь без дефиса.
- React Router нам пока избыточен — простой switch по статусу хватит. Полноценный роутинг введём в T-018, когда появятся каналы.
