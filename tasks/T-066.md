# T-066: Поиск по сообщениям (Postgres tsvector + GIN)

**Phase:** 4 — Уют
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~2 часа

## Контекст
Полнотекстовый поиск через postgres tsvector. Для 15-20 friends волне хватит — не нужны ElasticSearch / Meilisearch.

## Файлы в скоупе
- `packages/speedy/src/db/schema.ts` (добавить tsvector column в messages)
- `packages/speedy/drizzle/000X_search.sql` (миграция: добавить колонку + GIN индекс + trigger)
- `packages/speedy/src/routes/search.ts` (новый)
- `packages/polly/src/features/search/SearchScreen.tsx` (новый)
- `packages/polly/src/features/search/api.ts` (новый)

## Что сделать
1. **Schema migration**:
   - Добавить колонку `search_vector tsvector GENERATED ALWAYS AS (to_tsvector('russian', content)) STORED`.
   - Создать GIN индекс: `CREATE INDEX idx_messages_search ON messages USING GIN (search_vector)`.
   - Использовать `russian` configuration (или `simple` если хочется не зависеть от языка).
2. **`/api/search/messages`**:
   - query: `q` (текст), `channelId?` (ограничение), `authorId?`, `before?`, `after?` (дата), `limit?`
   - SQL: `WHERE search_vector @@ plainto_tsquery('russian', $q) AND (channel_id = $channelId OR $channelId IS NULL) ...`
   - ORDER BY `ts_rank_cd(search_vector, query) DESC` или по дате.
   - Limit 50 default.
   - Permission: фильтровать только каналы, к которым user имеет доступ (server membership).
3. **`SearchScreen.tsx`**:
   - Открывается через Ctrl+K или иконку лупы в шелле.
   - Input для query вверху, фильтры (channel, author, date) сбоку.
   - Список результатов: card per message, с context (preceding/following 1 line) и channel/author/date.
   - Highlight matched terms (через `ts_headline` в SQL или client-side через split-replace).
   - Клик → jump to message in channel.
4. **Tradeoffs**:
   - tsvector НЕ ищет внутри слов (`@вас` не найдёт `вася`). Если хочется — добавить `to_tsvector('russian', content)` + `to_tsquery('russian', $q || ':*')` для prefix-search. Это работает «как ожидается» для пользователя.

## DoD
- [ ] Поиск `привет` в 1000 сообщений выдаёт релевантные за < 100ms (с индексом)
- [ ] Поиск с фильтром по каналу — только из этого канала
- [ ] Результаты от чужих серверов (где user не member) — не возвращаются
- [ ] Search query на кириллице ищет с учётом морфологии (`играю` находит `игра`)
- [ ] `pnpm typecheck` чистый

## Out of scope
- Поиск по реакциям / прикреплённым файлам — нет.
- Fuzzy-search (опечатки) — нет, postgres не умеет из коробки.

## Подсказки
- `pg_trgm` extension даёт trigram-similarity для fuzzy. Можно потом, если будет нужно. По умолчанию — без него.
- Default `russian` configuration в postgres хороша, но не идеальна. Можно потом подключить `pg_search` или кастомные словари (фаза 5+).
- НЕ давай user'у писать raw `to_tsquery` syntax (со скобками и операторами) — используй `plainto_tsquery` или `websearch_to_tsquery` (последний поддерживает Google-style: кавычки, минус для исключения).
