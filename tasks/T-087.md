# T-087: Звонок и демо экрана в DM

**Phase:** 5 — Полировка
**Model:** Opus 4.7 · `high` (WebRTC, race conditions на 1-to-1 invite)
**Estimate:** ~4-5 часов

## Контекст

Сейчас в DM есть только текстовый чат — `DmScreen.tsx` не имеет action-кнопок в шапке. Дизайн (`designs/final-dm.jsx:170-188`) рисует три кнопки рядом с именем собеседника: «позвонить» (голос), «видео» (камера, требует T-082), «демо экрана».

LiveKit уже умеет всё, что нужно — звонок в DM это просто voice room с deterministic `room` name (`dm-${dmChannelId}`) и автоматическим invite-сигналом второй стороне через WS.

Читать: `tasks/T-030.md` (Guido), `tasks/T-031.md` (voice/join), `tasks/T-033.md` (useVoiceRoom), `tasks/T-064.md` (DM модель), `ARCHITECTURE.md` §3.2.

## Цель

В DM с любым собеседником можно начать voice-call, включить демо экрана, и второй стороне приходит invite-уведомление с возможностью принять или отклонить.

## Что сделать

1. **Backend room helper** в `guido.ts`:
   - `roomNameForDm(dmChannelId): 'dm-${id}'`. Стабильное имя — оба клиента попадают в одну комнату.
2. **`POST /api/voice/dm/:dmChannelId/join`**:
   - Проверка: user — один из двух participants DM. Иначе 403.
   - Выдача токена как в `T-031`, но с `room = dm-${dmChannelId}`.
   - Если в комнате ещё нет никого — broadcast `{t:'dm.call-invite', dmChannelId, fromUserId}` второй стороне.
   - Если вторая уже в комнате (она инициатор) — не шлём invite, просто join.
3. **WS events**:
   - `dm.call-invite` — отправляется только targeted userId (а не всем подписчикам канала). Расширить `broker` для personal-routing.
   - `dm.call-cancel` — инициатор отменил до того, как принято.
   - `dm.call-decline` — приёмник нажал «отклонить» (LiveKit участник удалён через `revokeUser`).
4. **DmScreen action buttons** (по дизайну `final-dm.jsx:170-188`):
   - Иконка «позвонить» (PhoneIcon) → `useVoiceRoom().joinDm(dmChannelId)`. Self-сторона сразу видит свой ParticipantTile, ждёт второго.
   - Иконка «видео» — заглушка с tooltip «нужна камера, см. T-082».
   - Иконка «демо» → join + сразу `startShare()`.
   - Все три **disabled** если уже в активном voice channel в другом месте.
5. **IncomingCall.tsx**:
   - Получает `dm.call-invite` через `wsClient.on`.
   - Toast в углу окна: «<displayName> звонит» + кнопки «принять» (→ join DM voice) и «отклонить» (→ POST cancel).
   - Звуковой сигнал (короткий `ring.mp3` в `public/`).
   - Auto-dismiss через 30 секунд (другая сторона получает timeout-cancel).
6. **useVoiceRoom — kind: 'dm'**:
   - Параметр `kind: 'channel' | 'dm'` для `joinChannel`.
   - При `'dm'` — токен от `/voice/dm/:id/join` вместо `/voice/:id/join`.
   - Layout в `VoiceScreen` тот же, но header показывает имя собеседника, не имя канала.

## Acceptance criteria (DoD)

- [ ] User-A кликает «позвонить» в DM → User-B видит toast «звонит» в углу окна
- [ ] User-B «принять» → оба в voice-комнате, аудио идёт
- [ ] User-B «отклонить» → у User-A toast «отклонил звонок»
- [ ] User-A отменяет звонок до ответа → у User-B toast пропадает
- [ ] Демо экрана в DM работает идентично server-каналу
- [ ] Второй DM (с третьим user'ом) во время активного звонка — кнопки disabled
- [ ] `pnpm typecheck` чистый
- [ ] Ручная проверка: на двух машинах звонок проходит за < 2 сек

## Out of scope

- Групповые звонки в DM (3+ человек) — нет, для этого voice-каналы серверов.
- История звонков (call log) — нет.
- Voicemail — нет.
- Видео с камеры — нужна T-082 (camera track в LiveKit).

## Подсказки

- LiveKit Room name для DM **должен** включать prefix `dm-` — иначе можно случайно зайти в room `dm-<uuid>` мимо permission-check. В `/voice/:channelId/join` запретить serverless room names ручной валидацией.
- Не используй WS-broadcast по `subscribedChannels` для invite — это публичный канал, а звонок privately targeted. Расширь `broker.publishToUser(userId, event)`.
- Auto-cancel при timeout: серверный setTimeout на 30s от момента invite. При фактическом join — clearTimeout.
- IncomingCall — это **не модалка** (mode='dialog'). Не блокируй ввод в чате; toast в углу.
