# T-033: Клиент — useVoiceRoom(channelId)

**Phase:** 2 — Голос
**Model:** Opus 4.7 · `high`
**Estimate:** ~3-4 часа

## Контекст
Главный хук для голосового канала. Подключение к LiveKit, mute/deafen, отслеживание активных спикеров, обработка disconnect. Это сложный кусок с правильной очисткой ресурсов — здесь Opus реально нужен.

Читать: LiveKit Client SDK docs (`livekit-client`), `final-voice.jsx` для понимания, какой state нужен UI.

## Файлы в скоупе
- `packages/polly/src/features/voice/useVoiceRoom.ts` (новый)
- `packages/polly/src/features/voice/store.ts` (новый — Zustand: state голоса)
- `packages/polly/src/features/voice/api.ts` (новый — join/leave REST)
- `packages/polly/src/lib/livekit.ts` (новый — singleton Room + хелперы)

## Что сделать
1. **deps**: `livekit-client`.
2. **`store.ts`** (Zustand):
   ```
   state:
     activeChannelId: string | null
     status: 'idle' | 'connecting' | 'connected' | 'reconnecting' | 'failed'
     muted: boolean
     deafened: boolean
     screenSharing: boolean
     participants: Map<userId, ParticipantState>
     activeSpeakers: Set<userId>
     error: string | null
   actions:
     joinChannel(channelId), leaveChannel(), toggleMute(), toggleDeafen()
   ```
3. **`lib/livekit.ts`**:
   - `getRoom(): Room` — singleton, создаётся лениво, реюзается между переподключениями.
   - `connectToChannel(token, url, room)` — обёртка над `room.connect(url, token)`, обработка ошибок.
   - `setupListeners(room, store)` — подписка на LiveKit events:
     - `ParticipantConnected`/`ParticipantDisconnected` → обновить store.participants.
     - `TrackSubscribed` (audio) → автоматически прикрепить к `<audio>` элементу (LiveKit умеет сам в Room API).
     - `TrackSubscribed` (video screen_share) → store.participants[id].videoTrack = track.
     - `ActiveSpeakersChanged` → store.activeSpeakers = новый set.
     - `Disconnected` → store.status='failed' с reason.
     - `ConnectionStateChanged` → отражение в status.
4. **`useVoiceRoom(channelId)`** хук:
   - При смене channelId → `joinChannel`.
   - `joinChannel`:
     1. Если уже подключены к другому каналу → `leaveChannel` (с очисткой!)
     2. POST /api/voice/:id/join → получаем token, url, room, snapshot
     3. Заполняем store.participants snapshot'ом
     4. `connectToChannel(token, url, room)`
     5. После connect — публикуем микрофонный трек (`room.localParticipant.setMicrophoneEnabled(true)`)
     6. store.status = 'connected'
   - `leaveChannel`:
     1. `room.disconnect()` — синхронно
     2. Очистка store (status='idle', participants=empty)
     3. Опционально: POST /api/voice/:id/leave (без await)
   - Cleanup в `useEffect`: при unmount или смене channelId — leave.
5. **Toggle actions**:
   - `toggleMute()` → `localParticipant.setMicrophoneEnabled(!muted)` + store update.
   - `toggleDeafen()` → mute + поставить volume=0 на все remote audio tracks. При undeafen — restore.
6. **Permissions** (Windows):
   - При первом `setMicrophoneEnabled(true)` браузер запросит permissions. WebView2 покажет prompt системы.
   - Если permission denied → store.error = 'no-mic-permission', UI показывает инструкцию.

## Подсказки
- LiveKit `Room` имеет встроенный auto-reconnect — НЕ пиши свой поверх. Просто слушай `ConnectionStateChanged`.
- При смене канала **никогда** не реюзай `Room` instance — `room.disconnect()` затем `new Room()`. Реюз приводит к state corruption в SDK.
- Деген mode: в dev можно подключиться к LiveKit двумя tabs одного браузера от одного user — LiveKit будет ругаться, что identity занят. Решение: identity = `${userId}:${windowId}`, где windowId уникален (crypto.randomUUID). Это для testing only — в проде один user = один identity.
- На Tauri WebView2 microphone permission persists между запусками, но НЕ через `getUserMedia` — через системную панель «Параметры → Конфиденциальность → Микрофон → Разрешить настольным приложениям». Если в первый раз не дать — придётся идти в системные настройки руками.
