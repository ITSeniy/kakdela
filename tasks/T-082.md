# T-082: Аудит-лог

**Phase:** 5
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~2 часа

## Контекст
Лог действий с правами админа: создания/удаления каналов, изменения ролей, бан/кик, инвайты. Для прозрачности в группе друзей — кто и что натворил.

## Файлы в скоупе
- `packages/speedy/src/db/schema.ts` (таблица `audit_log`)
- `packages/speedy/src/lib/audit.ts` (новый — helper для записи)
- `packages/speedy/src/routes/audit.ts` (новый — endpoint просмотра)
- Везде в существующих admin-эндпоинтах: вызов `audit.log(...)`.
- `packages/polly/src/features/settings/AuditLog.tsx` (новый)

## Что сделать
1. **Таблица `audit_log`**:
   - id, serverId, actorId, action enum (`channel.create`, `channel.delete`, `channel.update`, `member.promote`, `member.demote`, `member.kick`, `invite.create`, `invite.revoke`, `emoji.create`, `emoji.delete`).
   - targetType enum (`channel`, `user`, `invite`, `emoji`, `server`).
   - targetId uuid (nullable если delete).
   - metadata jsonb (старое и новое значение).
   - createdAt.
   - Index по (serverId, createdAt DESC).
2. **`audit.ts`** helper:
   - `await audit.log({ serverId, actorId, action, targetType, targetId, metadata })`.
   - Вызывать в каждом admin-эндпоинте (channels.create/update/delete, server-members PATCH/DELETE, invites POST/DELETE, emoji.create/delete).
3. **`GET /api/servers/:serverId/audit`** (только admin/owner):
   - query: `limit`, `before` (pagination).
   - Сразу резолвить actor и target в человеческие имена (JOIN).
   - Response: `[{ id, actor: User, action, target, metadata, createdAt }]`.
4. **`AuditLog.tsx`** в settings:
   - Список с фильтрами по action / actor / date.
   - Human-readable строки: «вася создал #общее 5 минут назад», «петя удалил вась из сервера».
5. **Не логировать**: сообщения (это спам), reactions, presence, voice события.

## DoD

- [ ] Не-admin не видит /audit endpoint (403)
- [ ] Пагинация работает
- [ ] Все admin-actions покрыты — проверь по grep'у в коде
- [ ] `pnpm typecheck` чистый

## Out of scope
- Экспорт лога в CSV — фаза 5+.
- Retention policy (старше N дней удалять) — фаза 5+.

## Подсказки
- Audit log пишется async (fire-and-forget) — не блокируй основной запрос на нём. Используй `setImmediate(() => audit.log(...))` или promise без await.
- metadata jsonb для diff'ов: `{ before: {...}, after: {...} }` — удобно для UI.
