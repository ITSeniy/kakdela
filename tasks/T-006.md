# T-006: Francine — drizzle config + первая миграция

**Phase:** 0 — Фундамент
**Model:** Sonnet 4.6 · `medium` reasoning
**Estimate:** ~1 час

## Контекст
- В скаффолде есть `packages/francine/src/index.ts` (заглушка) и `packages/speedy/drizzle/` (пустая).
- Цель — собрать минимальную CLI-команду `francine migrate`, которая поднимает схему БД на чистом postgres.
- Схема — только табличка `users` для начала (остальное добавим по мере фаз).

## Файлы в скоупе
- `packages/francine/package.json`
- `packages/francine/src/index.ts`
- `packages/speedy/src/db/schema.ts` (новый — drizzle table definitions)
- `packages/speedy/drizzle.config.ts` (новый)
- `packages/speedy/drizzle/0000_initial.sql` (сгенерится drizzle-kit'ом, но шапка нужна)

## Что сделать

1. **`speedy/src/db/schema.ts`**:
   ```ts
   import { pgTable, uuid, text, timestamp, pgEnum } from 'drizzle-orm/pg-core'

   export const userStatusEnum = pgEnum('user_status', ['online','idle','dnd','offline'])

   export const users = pgTable('users', {
     id: uuid('id').primaryKey().defaultRandom(),
     username: text('username').notNull().unique(),
     displayName: text('display_name').notNull(),
     email: text('email').notNull().unique(),
     passwordHash: text('password_hash').notNull(),
     avatarUrl: text('avatar_url'),
     status: userStatusEnum('status').notNull().default('offline'),
     customStatus: text('custom_status'),
     createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
     lastSeenAt: timestamp('last_seen_at', { withTimezone: true }).notNull().defaultNow(),
   })
   ```

2. **`speedy/drizzle.config.ts`**:
   ```ts
   import type { Config } from 'drizzle-kit'
   import 'dotenv/config'

   export default {
     schema: './src/db/schema.ts',
     out: './drizzle',
     dialect: 'postgresql',
     dbCredentials: { url: process.env.DATABASE_URL! },
     verbose: true,
     strict: true,
   } satisfies Config
   ```

3. **Сгенерировать миграцию**: запустить вручную `pnpm --filter @kakdela/speedy drizzle-kit generate`. Получится `drizzle/0000_xxx.sql`.

4. **`francine/src/index.ts`** — CLI:
   - Парсить первый аргумент: `migrate`, `seed`, `invite`, `help`.
   - `francine migrate`:
     - Подключиться к БД через `postgres-js`.
     - Использовать `drizzle-orm/postgres-js/migrator → migrate()`.
     - Указать путь к миграциям `../speedy/drizzle`.
   - `francine seed`: пока заглушка `console.log('TODO: seed')`.
   - `francine help`: список команд.

5. **`package.json` для francine**:
   ```json
   {
   ```

## Подсказки
- Drizzle-kit может ругаться на тип `pgEnum` — это нормально, всё генерится корректно.
- Если хочешь поглядеть на сгенерированную SQL — посмотри `drizzle/0000_*.sql`. Не правь руками; меняй `schema.ts` и перегенерируй.
- Не делай "migrate down" — drizzle их формально не поддерживает. Если что-то сломалось — `docker:reset` и накат заново.
