## Что сделать

1. **Зависимости** в `packages/speedy/package.json`:
   ```
   "fastify": "^5.x",
   "fastify-type-provider-zod": "^4.x",
   "@fastify/cors": "^10.x",
   "pino-pretty": "^11.x",
   "drizzle-orm": "^0.36.x",
   "postgres": "^3.x",
   "ioredis": "^5.x",
   "zod": "^3.x"
   ```
   Dev: `tsx`, `@types/node`, `typescript`.

2. **`env.ts`** — Zod-схема для всех переменных окружения. Падать при старте с понятным сообщением, если чего-то не хватает. Экспортировать тип `Env`.

3. **`logger.ts`** — pino. В dev — pino-pretty, в prod — json.

4. **`db.ts`** — клиент postgres.js + drizzle wrapper. Не выполнять запросов при импорте, только подготовка.

5. **`redis.ts`** — ioredis singleton с lazy connect.

6. **`routes/health.ts`** — `GET /healthz`:
   - Делает `SELECT 1` к postgres
   - Делает `PING` к redis
   - Возвращает `{ status: 'ok', db: 'ok' | 'fail', redis: 'ok' | 'fail', uptime: number }`
   - HTTP 200, если оба ОК; 503, если что-то упало.

7. **`index.ts`** — Fastify bootstrap:
   - Регистрация cors (только `PUBLIC_ORIGIN`)
   - Регистрация zod-type-provider
   - Регистрация `healthRoutes`
   - Graceful shutdown по SIGTERM/SIGINT
   - Listen на `SPEEDY_HOST:SPEEDY_PORT`

8. **Скрипты в package.json**:
   ```
   "dev":  "tsx watch src/index.ts",
   "build":"tsc",
   "start":"node dist/index.js",
   "typecheck":"tsc --noEmit"
   ```

## Подсказки
- Не используй `dotenv` явно — Node 20+ умеет `--env-file=.env` нативно, а Vite/Tauri умеют свой `.env`. Но в начале можно через `dotenv/config`, потом перейдём.
- Зачем zod для env: чтобы Claude Code не плодил `process.env.X!` с восклицательными знаками — это сразу ломает strict-mode.
