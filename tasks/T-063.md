# T-063: Attachments — files & images

**Phase:** 4 — Уют
**Model:** Opus 4.7 · `high` (security! magic bytes validation)
**Estimate:** ~3-4 часа

## Контекст
Прикреплять файлы к сообщениям. Базовая security: проверка размера, magic bytes (не доверять MIME), запрет исполняемых для безопасности (даже если друг по ошибке кинул .exe).

В T-053 уже сделан минимальный upload flow (presigned MinIO). Здесь — полноценная.

## Файлы в скоупе

- `packages/speedy/src/routes/files.ts` (расширение из T-053)

## Что сделать

### Backend

1. **Таблица `files`**:
   - id, ownerId, key (s3), originalName, contentType, sizeBytes, width, height (nullable, для картинок), createdAt
2. **`file-validation.ts`**:
   - Whitelist MIME: `image/jpeg`, `image/png`, `image/webp`, `image/gif`, `video/mp4`, `video/webm`, `audio/mp3`, `audio/ogg`, `application/pdf`, `text/plain`, `application/zip` — и **только**.
   - Max size: 25 MB по умолчанию (config через env).
   - Magic bytes check после upload: использовать пакет `file-type` для проверки реального типа из бинарника. Если не совпадает с заявленным contentType → удалить файл и вернуть ошибку.
3. **Эндпоинты**:
   - `POST /api/files/presign` — как в T-053, но теперь записывает в БД таблицу `files` со статусом `pending` и возвращает `fileId`.
   - `POST /api/files/:id/finalize` — клиент вызывает после успешного PUT в MinIO. Сервер:
     - Скачивает первые 4 KB из MinIO для magic bytes check.
     - Для картинок — определяет dimensions (через `sharp` или просто читая EXIF).
     - Обновляет файл status='ready', сохраняет width/height.
     - Если magic mismatch → DELETE объект в MinIO, удаляет запись, возвращает 422.
   - Messages POST принимает `attachments: string[]` (массив fileId) — связывает messages_files (через FK или JSON column).

### Frontend

4. **Composer attachments**:
   - Кнопка-скрепка → file picker.
   - Drag-n-drop на composer (visual «отпустите файл сюда» overlay при dragenter).
   - Paste из буфера (Ctrl+V → если image data в clipboard — добавить как attachment).
   - Max 10 файлов на сообщение.
5. **Preview перед отправкой**:
   - Под composer'ом — горизонтальная лента thumbnail'ов.
   - Прогресс upload'а (полоса 0-100%).
   - Иконка крестика для удаления из ленты.
6. **AttachmentView** в сообщении:
   - Image: thumbnail (max 400×300, sourced ?) + clicked → Lightbox.
   - Video: native `<video controls>`, max 600×400.
   - Audio: native `<audio controls>`.
   - Other: card-style тайл с иконкой типа файла, именем, размером, кнопкой download.
7. **Lightbox** — fullscreen для картинок:
   - Esc, click outside, X кнопка — закрыть.
   - Скачать original (не thumbnail).
   - Стрелки/scroll для перехода между картинками сообщения.

## DoD
- [ ] Drag-n-drop картинки в composer → preview → send → у другого user'а thumbnail + клик → lightbox
- [ ] Paste картинки из clipboard работает (snipping tool на Windows → Ctrl+V в composer)
- [ ] Загрузка .exe → 422 (magic bytes mismatch), файл не сохранён
- [ ] Файл > 25 MB → ошибка при presign
- [ ] PDF → отрисован как card с иконкой
- [ ] `pnpm typecheck` чистый

## Out of scope
- Inline видео-плеер с timestamps — нет.
- Превью PDF (рендер первой страницы) — нет.
- Image compression на клиенте перед upload — фаза 5.

## Подсказки
- `file-type` пакет умеет читать magic bytes из Buffer'а — не передавай весь файл, только первые ~50 байт хватит для большинства форматов.
- `sharp` для server-side image processing — мощный, но тяжёлый dep. Если задача — только узнать dimensions, попробовать сначала `probe-image-size` (читает только headers).
- Lightbox с keyboard navigation — внимание к focus trap. Использовать `react-focus-lock` если хочется быстро.
