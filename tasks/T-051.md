# T-051: Grid layout с screen-share tile'ами

**Phase:** 3 — Демонстрация экрана ⭐ MVP
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~2-3 часа

## Контекст
Дизайн в `designs/final-voice.jsx`. Когда кто-то демонстрирует экран, его tile становится **большим** (16:9), а avatar-tile'ы остальных — маленькие иконки сбоку или внизу. При нескольких одновременных демо — grid из больших tile'ов.

## Файлы в скоупе
- `packages/polly/src/features/voice/VoiceScreen.tsx` (расширение)
- `packages/polly/src/features/voice/ScreenTile.tsx` (новый)
- `packages/polly/src/features/voice/ParticipantTile.tsx` (доработка — режим compact)
- `packages/polly/src/features/voice/layout.ts` (новый — функция вычисления раскладки)

## Что сделать
1. **`layout.ts`** — pure функция:
   ```
   computeLayout(participants, screensharers): {
     focus: ScreenSource[],   // большие tile'ы для screen tracks
     dock: Participant[]       // маленькие avatar tiles
   }
   ```
   - Если 0 screen shares → все участники в основном grid (как в T-034)
   - Если 1+ screen shares → screens в основной области, все участники (включая шарящих) в маленьком dock'е внизу или сбоку
   - Layout focus area: 1 screen → fullscreen; 2 → split 50/50; 3-4 → 2×2; 5+ → 3×N
2. **`ScreenTile.tsx`** — большой 16:9 tile:
   - Live video через `track.attach()` 
   - Сверху overlay: имя демонстрирующего + иконка screen-share + RTT badge
   - Onhover: кнопки «full screen» (открывает в picture-in-picture или новом окне), «pin» (закрепить tile поверх остальных)
   - Click для fullscreen — через `element.requestFullscreen()`
3. **Dock** — горизонтальная полоска avatar-tile'ов внизу:
   - Кружки 48×48 с avatar, имя под кружком (truncated)
   - Активный спикер — зелёное кольцо
   - Mute crossed индикатор в углу
4. **Pin logic** (опционально):
   - User может «закрепить» один screen — он становится fullsize, остальные screens отправляются в dock как маленькие video-thumbs.
   - State в store: `pinnedScreenSourceId: string | null`.

## DoD
- [ ] Один user шарит экран → его screen big, все включая его — маленькие dock-tiles
- [ ] Двое шарят → split-screen 50/50, обе картинки видны
- [ ] Pin одного из screens → он на весь экран, второй уходит в dock (если pin реализован)
- [ ] Двойной клик по screen → fullscreen browser-режим
- [ ] При остановке демо tile корректно убирается, layout перестраивается

## Подсказки
- `track.attach()` возвращает `<video>` элемент с правильными `srcObject`. Не пытайся вручную создавать `<video>` и присваивать stream — LiveKit делает это правильнее.
- При unmount `<ScreenTile>` — обязательно `track.detach(element)` иначе будут утечки.
- CSS grid с `aspect-ratio: 16/9` на screen-tile + `object-fit: contain` на video — чтобы не растягивало.
- 60fps screen share — это много пикселей. Не делай дополнительной CSS-анимации на сам tile, только на UI поверх (overlay buttons).
