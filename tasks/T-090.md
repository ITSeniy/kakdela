# T-090: Кастомные роли (per-server теги в профиле)

**Phase:** 5 — Полировка
**Model:** Opus 4.7 · `high` (схема + permissions + UI admin)
**Estimate:** ~5-6 часов (большая задача)

## Контекст

Дизайн `designs/final-profile.jsx:101-124` рисует кастомные роли как цветные теги: «дизайн», «свой», «ранняя пташка». Сейчас в коде есть только три встроенные роли — `owner / admin / member` из `server_members.role`. Чтобы соответствовать дизайну, нужна полноценная система кастомных ролей.

Это **большая фича** для self-host на 15-20 друзей — можно сначала спросить, стоит ли. Эта карточка описывает реализацию, если решено делать.

Читать: `ARCHITECTURE.md` §6 (data model), `tasks/T-018.md` (member list), `tasks/T-068.md` (profile modal).

## Цель

Owner / admin сервера могут создавать кастомные роли с цветами; они отображаются в профиле каждого user'а и в member list.

## Что сделать

1. **Schema**:
   - `server_roles`: `id uuid pk, serverId fk, name text(32), color text (hex), position int, createdAt`. Уникальный индекс `(serverId, name)`.
   - `member_role_assignments`: `serverId fk, userId fk, roleId fk, assignedAt, assignedBy fk → users`. PK composite.
2. **Эндпоинты** (`/api/servers/:serverId/roles/*`, только admin/owner):
   - `GET /api/servers/:serverId/roles` — список.
   - `POST /api/servers/:serverId/roles` — `{ name, color }`. Валидация: name 1-32, color `^#[0-9a-f]{6}$`.
   - `PATCH /api/roles/:roleId` — переименовать / поменять цвет / переставить position.
   - `DELETE /api/roles/:roleId` — удалить + cascade убрать assignments.
   - `POST /api/roles/:roleId/members` `{ userId }` — назначить.
   - `DELETE /api/roles/:roleId/members/:userId` — снять.
3. **`UserProfile.roles`**: расширить тип — `Array<{ id, name, color, serverId }>`. В `GET /api/users/:id` подтянуть assignments **только по серверам, общим с requester'ом**.
4. **ProfileModal**: блок «роли · N» как в дизайне:
   - Pills с `border: 1px solid ${color}55`, цветная точка 6×6 слева, mono шрифт.
   - Если `isSelf` или requester — admin: после pills кнопка `+` → `AssignRoleModal`.
   - Группировать по `serverId`, если у user'а пересечения с requester'ом — несколько серверов.
5. **RolesEditor** (вкладка в server settings):
   - Список ролей сервера с drag-handle для position.
   - Создание / редактирование / удаление.
6. **AssignRoleModal**:
   - Список ролей текущего сервера + чекбоксы.
   - `POST` / `DELETE` при изменении.
7. **WS**: при assign/unassign — `{t:'role.assign', serverId, userId, roleId}` всем членам сервера, invalidate `user-profile` cache.
8. **CLI** в big-cheese:
   - `role create <serverId> <name> <color>`
   - `role assign <roleId> <userId>`
   - Для bootstrapping admin'а на пустом сервере.

## Acceptance criteria (DoD)

- [ ] Owner создаёт роль «дизайн» с цветом `#d68b6c` → она появляется в Roles list
- [ ] Owner назначает её Васе → у Васи в профиле виден pill «дизайн» с цветной точкой
- [ ] Удаление роли убирает её из всех профилей
- [ ] Member без admin прав не видит кнопку `+` в чужом профиле
- [ ] Роли отрисованы только по общим серверам (Вася не видит роли Пети из сервера, в котором его нет)
- [ ] WS broadcast: новый assignment — у других клиентов обновляется без F5
- [ ] `pnpm typecheck` чистый

## Out of scope

- Permission-роли (право создавать каналы, банить, и т.д.) — у нас `owner / admin / member` достаточно. Эти роли — **косметические** теги.
- Color picker advanced (HSV / dropper) — простой `<input type="color">` хватит.
- Drag-n-drop reorder между серверами — нет, роли scoped per server.
- Mention via `@<role>` — отдельная задача, если понадобится.

## Подсказки

- Не путай встроенные `owner / admin / member` (permissions) и кастомные roles (теги). Параллельные системы, не сливать.
- Cascade на удаление сервера: и `server_roles`, и `member_role_assignments` должны падать. Проверь FK constraints.
- Цвет хранить как hex string — это компактно и не требует токенов. Применять через inline `style={{ borderColor: `${role.color}55`, ...}}` — суффикс `55` это alpha ≈ 33%.
- Сначала покажи юзеру список новых задач и спроси, нужна ли вообще эта фича. Для 15-20 друзей теги могут быть просто украшением, а не required функционалом.
