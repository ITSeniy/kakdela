# T-022: Переключатель темы (persist + auto)

**Phase:** 1 — Текстовый дом
**Model:** Haiku 4.5 · `low`
**Estimate:** ~30 минут

## Контекст
Частично сделано в T-004 (заглушка). Здесь — нормальный store с persist и поддержкой системной темы.

## Файлы в скоупе
- `packages/polly/src/lib/theme.ts` (новый, или развернуть существующий)
- `packages/polly/src/components/ThemeToggle.tsx` (новый)
- `packages/polly/src/features/settings/ThemePicker.tsx` (новый — для будущего меню настроек)

## Что сделать
1. **`theme.ts`** (Zustand с persist):
   - State: `mode: 'light' | 'dark' | 'system'`
   - Action: `setMode(mode)`
   - При изменении вычисляется effective theme: если `system` — слушает `matchMedia('(prefers-color-scheme: dark)').matches`.
   - Effective тема выставляется через `document.documentElement.dataset.theme = ...`.
   - Persist в localStorage по ключу `kd:theme:mode`.
2. **`ThemeToggle.tsx`** (для шапки шелла или footer'а):
   - Кнопка с иконкой текущей темы (солнце / луна / монитор).
   - Клик циклит light → dark → system → light.
3. **`ThemePicker.tsx`** — три радио-кнопки. Будет использоваться в настройках профиля (T-068, фаза 5).
4. **Hot reload** при смене системной темы: подписаться на `matchMedia.addEventListener('change')` и обновлять effective.

- Анимированные переходы — позже.

## Подсказки
- Не используй Tailwind dark-mode `class` стратегию — у нас CSS-переменные через `data-theme`. Это сильнее (один и тот же класс `bg-kd-bg` автоматически адаптируется).
- В Tauri определение системной темы работает идентично браузеру через `matchMedia`. Никакой нативной интеграции через `@tauri-apps/plugin-os` не требуется.
