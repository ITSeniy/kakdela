# T-080: Threads (треды-обсуждения)

**Phase:** 5 — После бега в проде
**Model:** Opus 4.7 · `high`
**Estimate:** ~4-5 часов (большая задача)

## Контекст
Тред — это «канал в канале», начинается из конкретного сообщения. Сохраняется в БД как новый channel с `parentChannelId` и `parentMessageId`. UI — sidebar справа или fullscreen mode.

## Файлы в скоупе
- `packages/speedy/src/db/schema.ts` (расширить channels: parentChannelId, parentMessageId)
- `packages/speedy/src/routes/threads.ts` (новый)
- `packages/polly/src/features/threads/ThreadPanel.tsx` (новый — sidebar)
- `packages/polly/src/features/threads/ThreadList.tsx` (новый — список активных)
- `packages/polly/src/features/threads/api.ts` (новый)
- `packages/polly/src/features/chat/Message.tsx` (доработка — кнопка «начать тред», badge «N сообщений в треде»)

## Что сделать
1. **Schema**:
   - В channels добавить: `parentChannelId uuid nullable`, `parentMessageId uuid nullable`, `archivedAt timestamp nullable` (треды могут архивироваться).
   - Тред = channel с `kind='text'` + parentChannelId != null.
   - При удалении parent message → треды отвязываются, не удаляются (parentMessageId стираем).
2. **Эндпоинты**:
   - `POST /api/channels/:channelId/messages/:messageId/threads` → создаёт тред (новый channel с parentMessageId).
     - Имя треда — первые 50 символов parent сообщения (или user-provided).
   - `GET /api/channels/:channelId/threads` → активные треды в канале.
   - `POST /api/threads/:id/archive` → ставит archivedAt.
   - Messages — переиспользуют существующие endpoints, тред = просто channel.
3. **UI: Message → «начать тред»**:
   - В context menu сообщения «Начать тред».
   - Открывает inline-форму: input для названия (опц.) + textarea для первого сообщения.
   - При создании — тред появляется в правой панели, под parent сообщением — pill «👇 N сообщений в треде».
4. **ThreadPanel**:
   - Правый sidebar 320-400 px, поверх Member List (или toggle между ними).
   - Header: имя треда, кнопка «архивировать», крестик закрыть.
   - Body: список сообщений (как обычный chat, но компактнее).
   - Composer внизу.
5. **ThreadList**:
   - Под каналом в Channels list — экспандер «Threads (5)» → раскрывает список активных тредов.
   - Альтернатива: отдельный таб в Inbox (T-065).
6. **Уведомления**: упоминания в тредах работают одинаково с обычными mention'ами.

## DoD

- [ ] Сообщения в треде через WS приходят отдельно от родительского канала
- [ ] Badge «N сообщений» обновляется на parent сообщении
- [ ] Архивация треда — он пропадает из active list но доступен через filter
- [ ] `pnpm typecheck` чистый

## Out of scope
- Воut-of-band threads (без parent сообщения) — нет.
- Forum-style каналы (где весь канал — список тредов) — нет.

## Подсказки
- Треды наследуют permissions parent канала. Не делай отдельную систему permissions.
- Когда тред активен — клиент должен подписаться на оба channelId (родительский и тред). Расширь subscribedChannels logic в WS gateway.
