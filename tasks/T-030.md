# T-030: Guido — выдача LiveKit-токенов

**Phase:** 2 — Голос
**Model:** Opus 4.7 · `high`
**Estimate:** ~2 часа

## Контекст
LiveKit использует JWT-токены с конкретным room name и permissions. Сервер их выдаёт, клиент использует для подключения. «Guido» в нашей кодовой схеме — модуль внутри speedy, отвечающий за LiveKit (по образу одноимённого пицца-кота). Отдельный сервис не делаем — это просто слой кода.

Читать: `ARCHITECTURE.md` §3.2 (про guido), §4 про webhook events. Документация LiveKit: server-sdk-js → `AccessToken`.

## Файлы в скоупе
- `packages/speedy/src/media/guido.ts` (новый)
- `packages/speedy/src/media/types.ts` (новый — VoiceToken, VoiceRoom)
- `packages/speedy/src/env.ts` (использовать LIVEKIT_*)

## Что сделать
1. **deps**: `livekit-server-sdk`.
2. **`guido.ts`** — функциональный модуль:
   - `issueToken({ userId, channelId, displayName, canPublish=true, canSubscribe=true, canPublishData=true })` → `{ token: string, url: string, room: string }`
     - room name = `voice-${channelId}` (детерминированно — все попадают в одну комнату на канал)
     - TTL токена: 6 часов (если человек висит дольше — переподключится)
     - metadata токена: `JSON.stringify({ userId })` — пригодится в webhook'ах
     - `identity` = `userId` (НЕ displayName — username должен быть стабильным ID)
     - `name` = `displayName` (это то что LiveKit отдаёт другим участникам)
   - `revokeUser({ userId, channelId })` → вызов LiveKit Admin API `removeParticipant`
   - `listParticipants(channelId)` → `RoomService.listParticipants(room)`, возвращает массив `{ userId, displayName, joinedAt, isPublishing, isScreenSharing }`
3. **Не делаем абстракцию над SFU** — LiveKit foreseeable single source of truth, никакого `interface MediaServer` пока не нужно.
4. **Тестовый эндпоинт** `GET /api/internal/livekit-debug` (только в `NODE_ENV=development`): возвращает выданный токен для test-user — чтобы можно было проверить через `livekit-cli`.

## Подсказки
- `livekit-server-sdk` имеет два класса: `AccessToken` для генерации и `RoomServiceClient` для админских операций. Не путай.
- TTL передаётся в секундах в `addGrant`, а не в `new AccessToken` — там это только настройка.
- LiveKit URL для клиента и для сервера может различаться (через Caddy reverse-proxy в проде). Передавай `LIVEKIT_URL` обе стороны через токен (отдельным полем в ответе клиенту).
