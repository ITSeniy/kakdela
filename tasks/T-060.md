# T-060: Реакции на сообщения

**Phase:** 4 — Уют (после MVP)
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~2-3 часа

## Контекст
Эмодзи-реакции на сообщения. В дизайне `final-chat.jsx` они уже отрисованы как пилюли под сообщением. Используем стандартный набор Unicode эмодзи через `emoji-mart` или похожую библиотеку. Custom emoji — отдельная задача T-081.

## Что сделать
1. **Таблица `reactions`**:
   - `messageId` fk
   - `userId` fk
   - `emoji` text (unicode codepoint, например '👍' или ':custom-id:' для T-081)
   - `createdAt`
   - Composite PK: (messageId, userId, emoji) — нельзя реагировать тем же эмодзи дважды.
2. **Эндпоинты**:
   - `POST /api/messages/:id/reactions` body `{ emoji }` → создаёт реакцию или 200 если уже есть (идемпотентность).
   - `DELETE /api/messages/:id/reactions/:emoji` → удаляет.
   - При создании/удалении — WS broadcast.
3. **WS events**:
   - `{t:'reaction.add', channelId, messageId, userId, emoji}`
   - `{t:'reaction.remove', channelId, messageId, userId, emoji}`
4. **Агрегация в API сообщений**:
   - При GET /messages — JOIN на reactions с COUNT + ARRAY_AGG(userId).
   - Структура в Message: `reactions: { emoji: string, count: number, users: string[] }[]`
   - Это позволяет рендерить «👍 3» и показать кто на hover.
5. **`Reactions.tsx`**:
   - Принимает `reactions: ReactionAggregate[]` и текущего userId.
   - Рендерит пилюли с emoji + count.
   - Подсвечивает пилюли, в которых user сам реагировал.
   - Клик по пилюле: toggle (add/remove своей реакции).
   - При hover на пилюле — popover «вася, петя, лена».
   - Кнопка «+» справа → открывает EmojiPicker.
6. **`EmojiPicker.tsx`**:
   - Использовать `emoji-mart` или `@emoji-mart/react` (lightweight, есть категоризация и поиск).
   - Поиск по shortcode (`:smile`) + рекент list (localStorage `kd:emoji:recent`).

- Animated emoji — нет.

## Подсказки
- Unicode эмодзи занимают 1-4 codepoints — не используй `length` для подсчёта, а `[...emoji].length` если надо.
- emoji-mart содержит ~12000 эмодзи и весит ~3 MB JSON — лениво грузить через dynamic import только когда picker открывается.
