# T-067: Web Push нотификации

**Phase:** 4 — Уют (опционально)
**Model:** Opus 4.7 · `high`
**Estimate:** ~3-4 часа

## Контекст
**Внимание**: с переходом на Tauri desktop этот таск становится менее приоритетным. Для desktop-приложения нативные нотификации (T-086) лучше Web Push. Эту задачу имеет смысл делать только если:
- Хочешь нотификации и в `pnpm dev:web` режиме (browser fallback).
- Планируешь PWA-вариант в будущем.

**Если приоритеты ограничены — пропустить эту задачу, перейти к T-086** (system tray + native notifications).

Этот таск задокументирован для полноты, помечен как **опциональный**.

## Файлы в скоупе
- `packages/speedy/src/db/schema.ts` (таблица `push_subscriptions`)
- `packages/speedy/src/routes/push.ts` (новый)
- `packages/speedy/src/notify/web-push.ts` (новый)
- `packages/polly/public/sw.js` (service worker — только для web mode)
- `packages/polly/src/features/notify/registerPush.ts` (новый)

## Что сделать
1. **VAPID keys** генерация: `npx web-push generate-vapid-keys` — записать в env (`VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT='mailto:admin@...'`).
2. **`push_subscriptions`** таблица: userId, endpoint, p256dh, auth, userAgent, createdAt.
3. **`POST /api/push/subscribe`** — клиент сохраняет subscription. `DELETE /api/push/subscribe` — отписка.
4. **Service Worker** (`sw.js`): обработчик `push` event → `self.registration.showNotification(title, options)`.
5. **`registerPush.ts`**: только в web mode (когда `'serviceWorker' in navigator && !isTauri`). Регистрирует SW + запрашивает permission + сохраняет subscription.
6. **`web-push.ts`** в speedy: helper для отправки push на конкретного user'а. Триггеры:
   - `@mention` → push «вася упомянул вас в #общем: ...».
   - DM message → push «вася: ...».
   - Не пушить если user активен (`presence.online`) или открыт нужный канал.

## Out of scope
- iOS Safari (PWA push) — не поддерживается до iOS 16.4+.
- Persistent notifications (с actions) — упрощаем.

## Подсказки
- pacakge `web-push` (Node) — почти 10 лет, стабильный.
- В Tauri 2 теоретически service worker работает в WebView2, но не имеет смысла — нативные нотификации лучше.
