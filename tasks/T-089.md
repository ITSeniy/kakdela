# T-089: Профиль — часовой пояс и «о себе»

**Phase:** 5 — Полировка
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~1.5-2 часа

## Контекст

Дизайн `designs/final-profile.jsx` рисует два блока, которых нет в реализации:
- **«о себе»** (`final-profile.jsx:68-78`) — биография 1-3 строк.
- **«часовой пояс»** (`final-profile.jsx:91-99`) — IANA timezone + текущее время в нём, формат «МСК · 11:24».

Это два простых поля в `users` и их UI-вывод. После добавления `ProfileModal` и `ProfileEditForm` будут визуально соответствовать дизайну.

## Файлы в скоупе

- `packages/speedy/src/db/schema.ts` (`users.about text nullable`, `users.timezone text nullable`)
- `packages/speedy/drizzle/000X_profile_fields.sql` (миграция)
- `packages/speedy/src/routes/users.ts` (вернуть about и timezone в `GET /api/users/:id`)
- `packages/speedy/src/routes/me.ts` (принимать оба поля в `PATCH /api/me`)
- `packages/ginzu/src/api-types.ts` (расширить `UserProfile`)

- `packages/polly/src/features/profile/ProfileEditForm.tsx` (два input'а)

## Цель

В профиле любого user'а виден текст «о себе» и его часовой пояс с текущим временем; свой можно редактировать.

## Что сделать

1. **Schema** — добавить две колонки в `users`:
   - `about text nullable` (max 280 символов на app-уровне — Zod).
   - `timezone text nullable` (IANA, например `Europe/Moscow`).
2. **Migration**: nullable + дефолт NULL. Без backfill.
3. **`UserProfile` тип** в ginzu: добавить `about: string | null`, `timezone: string | null`.
4. **`PATCH /api/me`** принимает `about?: string`, `timezone?: string`. Валидация:
   - `about`: trimmed, ≤ 280, или null для очистки.
   - `timezone`: проверка через `Intl.DateTimeFormat(undefined, { timeZone: tz })` — конструктор кидает RangeError на невалидной зоне. Ловим, возвращаем 400.
5. **ProfileModal — блок «о себе»**: если `profile.about` — карточка `bg-kd-panel-alt` + uppercase mono-label + текст. Если пусто и `isSelf` — placeholder «расскажи о себе...» с прозрачным фоном; не свой — блок скрыт.
6. **ProfileModal — блок «часовой пояс»**: если `profile.timezone` — карточка с label «ЧАСОВОЙ ПОЯС» и значением вида «МСК · 11:24», где:
   - abbr — `Intl.DateTimeFormat('ru', { timeZone: tz, timeZoneName: 'short' }).formatToParts(...)` → найти `timeZoneName` part.
   - время — `Intl.DateTimeFormat('ru', { timeZone: tz, hour: '2-digit', minute: '2-digit' }).format(now)`.
   - Обновлять время раз в минуту через `useEffect` с `setInterval(60_000)` (только пока модалка открыта).
7. **ProfileEditForm**:
   - Поле `about` — `<textarea rows={3} maxLength={280}>` с счётчиком символов.
   - Поле `timezone` — `<select>` с короткой подборкой популярных IANA-зон + опция «авто» (= `Intl.DateTimeFormat().resolvedOptions().timeZone`).
   - При save передаются в `patchMe`.

## Acceptance criteria (DoD)

- [ ] Поля `about` и `timezone` сохраняются и читаются через API
- [ ] Установка невалидной timezone (например `Mars/Tharsis`) → 400 от backend
- [ ] При закрытии и открытии модалки текущее время в часовом поясе обновлено
- [ ] Очистка «о себе» (отправка пустой строки) → поле в БД становится NULL
- [ ] Блоки скрыты у пользователя без `about` / `timezone`, если это не свой профиль
- [ ] `pnpm typecheck` чистый

## Out of scope

- Поля «локация» / «pronouns» / «соцсети» — самохост, не нужно.
- Автодетект timezone при регистрации — пользователь сам выберет.
- Подсветка «сейчас, наверное, спит» (вне рабочих часов) — overkill.

## Подсказки

- `Intl.DateTimeFormat` доступен во всех браузерах. Не подтаскивай `moment-timezone` или `luxon` ради этого.
- `timeZoneName: 'short'` может вернуть «GMT+3» вместо «МСК». Это нормально — fallback в код.
- 280 символов — не случайно: укладывается в 2-3 строки UI и совпадает с лимитом твиттера, который пользователи интуитивно знают.
