# T-088: DM welcome card («начало переписки»)

**Phase:** 5 — Полировка
**Model:** Haiku 4.5 · `low`
**Estimate:** ~30-45 минут

## Контекст

В пустом DM (когда ни одного сообщения ещё нет) дизайн `designs/final-dm.jsx:196-207` рисует welcome-блок: «это начало вашей переписки с <name>. вы оба в комнатах <X>, <Y>, <Z>». Сейчас при открытии пустого DM пользователь видит пустую область — нет ясного «empty state».

## Файлы в скоупе

- `packages/polly/src/features/dm/DmScreen.tsx` (отрисовать welcome card если `messages.length === 0`)
- `packages/polly/src/features/profile/api.ts` (если нужен — уже есть `getUserProfile` с `sharedServers`)

## Цель

При открытии DM без сообщений показать тёплый empty-state с напоминанием о контексте знакомства.

## Что сделать

1. **DmScreen**: после `useMessages(dmChannelId)`, если первая страница пустая и нет pending — рендер `<DmWelcomeCard otherUser={otherUser} sharedServers={otherUser.sharedServers}/>` **над** Composer'ом.
2. **DmWelcomeCard.tsx** (новый, в `features/dm/`):
   - Большой avatar собеседника по центру (size=72).
   - Заголовок: «это начало вашей переписки с **<displayName>**».
   - Если `sharedServers.length > 0` — мелким шрифтом: «вы оба в <Server1>, <Server2>». Не более трёх имён, остальное «+ N».
   - Если пересечений нет (можно и такое — DM из inbox по mention) — текст «вас тут особо ничего не связывает кроме теплого чая. ну, давайте уже».
   - Никаких CTA — это пассивный блок, скрывается с первым отправленным сообщением.
3. **Получение `sharedServers`**: переиспользовать `getUserProfile(otherUserId)` (уже есть). Кэшировать через `useQuery({ queryKey: ['user-profile', otherUserId], staleTime: 60_000 })` — DM-партнёр сравнительно стабилен, лишнего трафика не будет.

## Acceptance criteria (DoD)

- [ ] Открыть новый DM (с user'ом, с которым ещё не писали) → виден welcome-блок
- [ ] Отправить первое сообщение → блок исчезает
- [ ] Если оба в одном сервере «Друзья и кофе» — блок упоминает его
- [ ] Если общих серверов нет — блок показывает fallback-текст
- [ ] При reload страницы пустого DM блок снова виден (без flicker — лучше всего через initial state в `useQuery`)
- [ ] `pnpm typecheck` чистый

## Out of scope

- «Кто вас познакомил» (mutual friends) — у нас all-friends-by-default, не имеет смысла.
- Анимация исчезновения блока — лишнее.

## Подсказки

- Empty-state в `MessageList` уже есть для случая «загружается»; не сломай его. Welcome card живёт **вне** MessageList, отдельным компонентом в DmScreen.
- Не показывай welcome пока `isLoading === true` — иначе будет моргание при первой загрузке.
