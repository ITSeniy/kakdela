# T-035: Push-to-talk и voice-activated в настройках

**Phase:** 2 — Голос
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~2 часа

## Контекст
Два режима микрофона:
- **Voice activated** (по умолчанию): микрофон всегда «открыт», LiveKit использует встроенный VAD.
- **Push-to-talk**: микрофон mute'нут, пока user не нажмёт hot-key (по умолчанию Space, настраиваемо).

В Tauri глобальные hot-keys (работающие при свернутом окне) делаются через `tauri-plugin-global-shortcut`. Без него — только при сфокусированном окне приложения через keydown listener.

## Что сделать
1. **Store**:
   - `inputMode: 'voice-activated' | 'push-to-talk'` (persist в localStorage)
   - `pttKey: string` (default 'Space', persist)
   - `pttHolding: boolean` (только в memory)
2. **`usePushToTalk()`** хук:
   - Если `inputMode === 'push-to-talk'`:
     - keydown/keyup listeners на window (в-фокус режим).
     - При keydown с матчем pttKey → `pttHolding=true` + `setMicrophoneEnabled(true)`.
     - При keyup → `pttHolding=false` + `setMicrophoneEnabled(false)`.
   - Если `inputMode === 'voice-activated'`: микрофон управляется только из toggleMute, hot-key не работает.
3. **Global shortcut через Tauri** (опционально, можно отложить):
   - В фокусе достаточно window listener'ов. Если хотим работать когда окно свёрнуто — `globalShortcut.register('Space', handler)` через Tauri плагин. **Важно**: пробел как global shortcut будет конфликтовать со всем подряд (любая текстовая страница) — лучше дефолт сделать `RAlt` или `RShift`, и предупредить user'а.
4. **`VoiceSettings.tsx`** — компонент с радио input mode и input для key binding. UI для T-068, но логика рабочая.
   - Pressed key захват: компонент в «recording» режиме, ловит next keydown, сохраняет.
5. **Визуальная индикация PTT**:
   - В `VoiceControls`: если PTT и `pttHolding=false` → микрофонная кнопка с серым фоном + текст «зажмите [Space]».
   - Если `pttHolding=true` → зелёный фон, «вы говорите».

## Подсказки
- `KeyboardEvent.code` использовать вместо `key` — `code` стабилен относительно раскладки (KeyV = физическая V в любой раскладке).
- При `inputMode = 'push-to-talk'` НЕ давай также `toggleMute` работать — это путает. Логика: toggleMute доступен только в voice-activated.
- LiveKit `setMicrophoneEnabled` имеет ~100ms задержку на actually enable/disable трека. Для PTT это OK, но user может потерять первые 100ms речи. Решение: не публиковать/анпубликовать трек, а делать `track.mute()` / `track.unmute()` — гораздо быстрее.
