# T-011: Инвайт-коды

**Phase:** 1 — Текстовый дом
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~2 часа

## Контекст
В КакДеле нет открытой регистрации — только по инвайту. Это базовый антиспам для самохоста (на 15-20 друзей нет смысла в email-верификации, инвайт-кода достаточно).

## Файлы в скоупе

- `packages/speedy/src/routes/invites.ts` (новый)
- `packages/francine/src/index.ts` (команда `invite create`)
- `packages/ginzu/src/api-types.ts`

## Что сделать
1. **Таблица `invites`**:
   - `code` text pk (8 символов, base32 без неоднозначных: `abcdefghjkmnpqrstuvwxyz23456789`)
   - `serverId` uuid fk → servers.id
   - `createdBy` uuid fk → users.id (nullable — `null` для инвайтов от admin CLI)
   - `expiresAt` timestamp (nullable)
   - `maxUses` int (nullable — `null` = ∞)
   - `useCount` int default 0
   - `revoked` boolean default false
2. **Эндпоинты**:
   - `POST /api/servers/:serverId/invites` — создаёт инвайт. Требует роль admin/owner. Тело: `{ expiresInDays?, maxUses? }`. Возвращает `{ code, url }`.
   - `GET /api/invites/:code` — публично (для онбординга). Возвращает `{ serverName, serverIcon, expiresAt }` или 404. Не раскрывает `createdBy`.
   - `POST /api/invites/:code/accept` — требует auth (используется после регистрации). Атомарно увеличивает useCount, добавляет в `server_members`. Падает 410 если `revoked`/expired/maxUses достигнут.
   - `DELETE /api/invites/:code` — owner/admin может отозвать.
3. **CLI**: `pnpm francine invite create --server <id> [--expires-in 7d] [--max-uses 5]` создаёт инвайт без auth (для bootstrapping первого пользователя).
4. **Генерация кода**: `crypto.randomBytes(5)` → base32 → 8 символов. Проверять уникальность в цикле (вероятность коллизии ничтожна на 15-20 человек, но цикл с 3 попытками не помешает).

## DoD
- [ ] `francine invite create --server <id>` печатает код вида `kj4m9pq2`
- [ ] `GET /api/invites/kj4m9pq2` без auth возвращает данные сервера
- [ ] `POST /api/invites/kj4m9pq2/accept` с истёкшим инвайтом → 410 с кодом `invite-expired`
- [ ] Параллельный accept одним инвайтом с `maxUses: 1` не позволяет двоим войти (тест: race в двух терминалах)
- [ ] `pnpm typecheck` чистый

## Подсказки
- Гонку на `useCount` решает `UPDATE ... WHERE code=$1 AND use_count < max_uses RETURNING ...` — атомарно в одном SQL.
- В дизайне `final-onboarding.jsx` ввод кода — без дефисов и пробелов. Принимать и `kj4m9pq2`, и `kj4m-9pq2` (стрипать non-alphanum) — друзья будут копировать как попало.
