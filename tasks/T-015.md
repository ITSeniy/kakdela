# T-015: REST — список серверов, детали, каналы

**Phase:** 1 — Текстовый дом
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~1.5-2 часа

## Контекст
Базовые read-эндпоинты для шелла приложения. Все требуют auth и проверяют членство пользователя.

## Файлы в скоупе
- `packages/speedy/src/routes/servers.ts` (новый)
- `packages/speedy/src/routes/channels.ts` (новый)
- `packages/ginzu/src/api-types.ts` (схемы ответов)

## Что сделать
1. **`GET /api/servers`** — все серверы, в которых состоит текущий user. Возвращает `Server[]` (id, name, iconUrl).
2. **`GET /api/servers/:serverId`** — детали + список каналов. Возвращает `{ server, channels, memberCount }`. Проверка: user должен быть в `server_members`, иначе 403.
3. **`GET /api/servers/:serverId/members`** — список членов с базовой инфой (id, displayName, avatarUrl, status, role). Сортировка: онлайн сначала, потом по displayName.
4. **`GET /api/channels/:channelId`** — детали канала. Проверка: user — член сервера, к которому относится канал.
5. **`POST /api/servers/:serverId/channels`** (только admin/owner) — создать канал. Тело: `{ name, kind, category?, topic? }`.
6. **`PATCH /api/channels/:channelId`** (admin/owner) — переименовать, поменять topic, position.
7. **`DELETE /api/channels/:channelId`** (admin/owner) — удалить. Hard delete (сообщения тоже).
8. **Хелпер** `assertMember(userId, serverId)` и `assertRole(userId, serverId, roles: Role[])` в `lib/permissions.ts`. Использовать везде.

## DoD
- [ ] Все эндпоинты валидируются Zod-схемами
- [ ] Запрос к чужому серверу → 403 с кодом `forbidden`
- [ ] Несуществующий ID → 404
- [ ] `pnpm typecheck` чистый
- [ ] Ручная проверка через `curl` с access-токеном — все эндпоинты отвечают ожидаемой структурой

## Out of scope
- Голосование каналов / threads / category-таблица — потом.
- Soft delete — пока hard.

## Подсказки
- При удалении канала FK на messages должен быть `ON DELETE CASCADE`, иначе будет ошибка. Проверь в schema.ts.
- `members` ответ — не выгружай password_hash случайно. Используй явный SELECT с whitelist полей или drizzle-query с `columns: { ... }`.
