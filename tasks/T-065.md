# T-065: Inbox — упоминания и непрочитанные

**Phase:** 4 — Уют
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~2 часа

## Контекст
Дизайн в `designs/final-inbox.jsx`. Это global view всех @упоминаний и непрочитанных threads (когда threads появятся в T-080). На MVP-расширении — только упоминания.

## Файлы в скоупе
- `packages/speedy/src/db/schema.ts` (таблица `mentions`)
- `packages/speedy/src/lib/mention-extractor.ts` (новый — парсинг)
- `packages/speedy/src/routes/messages.ts` (доработка — при POST/PATCH парсим @mentions)
- `packages/speedy/src/routes/inbox.ts` (новый)
- `packages/polly/src/features/inbox/InboxScreen.tsx` (новый)

## Что сделать
1. **Таблица `mentions`**:
   - id, messageId fk, mentionedUserId fk, mentionType enum('user','everyone','here'), readAt timestamp nullable.
2. **`mention-extractor.ts`**:
   - Принимает текст сообщения, parses `@username` или `@<userId>` syntax.
   - Returns: `Array<{ userId, type }>`.
   - Также распознаёт `@everyone` (требует роль admin/owner — иначе ignore), `@here` (online members).
3. **Хук в POST/PATCH /messages**:
   - После insert сообщения — extract mentions, insert в `mentions`.
   - На PATCH — diff: новые добавить, удалённые убрать (по pair messageId+userId).
4. **WS event**: `{t:'mention', messageId, channelId, mentionedUserId}` отдельно отправляется упомянутому, для красного бейджа в нативном tray (T-086).
5. **`/api/inbox/mentions`**: возвращает страничку непрочитанных упоминаний для текущего user'а, с context (message + channel + author).
6. **`POST /api/inbox/mentions/read`** body `{ messageIds }` — массово помечает прочитанными.
7. **`InboxScreen.tsx`**:
   - Reachable через клик-иконку в углу шелла (или в server rail).
   - Tab «упоминания» (default) — список mentions с jump-to-message при клике.
   - При render — auto-mark прочитанными те, что в viewport (через IntersectionObserver).
   - Empty state: «всё спокойно, никто вас не упоминал».

## DoD
- [ ] `@вася` в сообщении → у Васи в Inbox появляется
- [ ] Клик в Inbox → переход на канал + scroll к сообщению + flash highlight
- [ ] `@everyone` от member'а игнорируется (silent), от owner'а — отправляется всем
- [ ] Прочитанные пропадают через 5 секунд после рендера (или сразу при клике)
- [ ] Edit сообщения с добавленным @user — у user'а появляется новое
- [ ] `pnpm typecheck` чистый

## Out of scope

## Подсказки
- Парсинг mentions может конфликтовать с email-like `something@example.com` в тексте. Решение: требовать start-of-line или whitespace перед `@`, и лучше использовать ID-формат `@<uuid>` в сериализованном тексте (как Discord делает с `<@123456>`).
- Не отправляй mention себе самому — отфильтровать.
