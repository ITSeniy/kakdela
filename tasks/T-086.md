# T-086: Системный трей + native notifications

**Phase:** 5 — Desktop polish
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~3 часа

## Контекст
Tauri 2 имеет `tray-icon` и `notification` плагины. Здесь делаем:
- Иконка в трее с меню (toggle window, mute, status, quit).
- Сворачивание в трей при закрытии окна (а не quit — это desktop convention).
- Native notifications на @mention и DM.
- Badge с unread count в trayicon.

## Файлы в скоупе
- `packages/polly/src-tauri/Cargo.toml` (плагины tray-icon, notification)
- `packages/polly/src-tauri/src/lib.rs` (setup tray + handlers)
- `packages/polly/src-tauri/capabilities/default.json` (permissions)
- `packages/polly/src/lib/host/tray.ts` (новый — JS API)
- `packages/polly/src/lib/host/notify.ts` (новый — JS API)
- `packages/polly/src/features/notify/triggers.ts` (новый — WS handlers → notifications)

## Что сделать
1. **Cargo.toml**: добавить `tauri-plugin-notification = "2"` и (если в Tauri 2 не встроено) `tauri-plugin-tray-icon`. Tauri 2 имеет tray-icon в core, не как отдельный плагин — проверь.
2. **`src-tauri/src/lib.rs`**:
   ```rust
   use tauri::{
     menu::{Menu, MenuItem},
     tray::TrayIconBuilder,
     Manager,
   };
   
   tauri::Builder::default()
     .plugin(tauri_plugin_notification::init())
     .setup(|app| {
       let quit_i = MenuItem::with_id(app, "quit", "Выход", true, None::<&str>)?;
       let show_i = MenuItem::with_id(app, "show", "Показать", true, None::<&str>)?;
       let menu = Menu::with_items(app, &[&show_i, &quit_i])?;
       
       TrayIconBuilder::new()
         .menu(&menu)
         .tooltip("как дела")
         .icon(app.default_window_icon().unwrap().clone())
         .on_menu_event(|app, event| match event.id.as_ref() {
           "quit" => app.exit(0),
           "show" => {
             if let Some(window) = app.get_webview_window("main") {
               let _ = window.show();
               let _ = window.set_focus();
             }
           }
           _ => {}
         })
         .build(app)?;
       Ok(())
     })
     // ... остальное
   ```
3. **Поведение на close**:
   - В `tauri.conf.json` window: `"hideOnClose": true` (Tauri 2 имеет это? проверь). Иначе через `window.on_close_requested` handler — `event.preventDefault()` и `window.hide()`.
4. **Notifications JS API**:
   ```ts
   import { isPermissionGranted, requestPermission, sendNotification } from '@tauri-apps/plugin-notification'
   
   export async function notify(title: string, body: string, onClick?: () => void) {
     if (!await isPermissionGranted()) {
       const p = await requestPermission()
       if (p !== 'granted') return
     }
     sendNotification({ title, body })
     // onClick handling — через listening events
   }
   ```
5. **Триггеры**:
   - WS `mention` event + (user not in this channel OR window not focused) → notify.
   - WS `msg.new` в DM + не открыт этот DM → notify.
   - Click on notification → focus window + navigate to channel.
6. **Badge с unread**:
   - На Tauri 2 — `tray.set_title("3")` для текстового badge поверх иконки. Или динамически generate icon с number.
   - На macOS dock есть `setBadgeCount` API — но не наша платформа.

## Подсказки
- Tray icon с badge — в Tauri 2 нет встроенного API. Workaround: генерировать PNG с цифрой и подставлять через `set_icon`.
- Уведомления требуют permission на Win10/11. При первом запросе Windows покажет toast с кнопкой «разрешить».
- Не спамь notifications — debounce: одна нотификация на канал в минуту. Если за минуту 5 mentions — одна нотификация «у вас 5 сообщений в #общем».
