# T-083: Несколько серверов — создание / переключение / приглашение

**Phase:** 5
**Model:** Sonnet 4.6 · `medium`
**Estimate:** ~2-3 часа

## Контекст
До этого момента у нас один дефолтный сервер. Здесь — UI для создания и присоединения к новым серверам. В контексте 15-20 друзей это будет редкая фича (зачем им разные серверы?), но иногда нужно — отдельный «work» и «play» серверы.

- `packages/polly/src/features/servers/CreateServerModal.tsx` (новый)
- `packages/polly/src/features/servers/JoinServerModal.tsx` (новый)
- `packages/polly/src/features/servers/InviteModal.tsx` (новый)

- `packages/polly/src/features/onboarding/WelcomeScreen.tsx` (новый — мульти-сервер welcome, переехал из T-013)

## Что сделать
1. **`POST /api/servers`** (auth):
   - Body: `{ name, iconUrl? }`.
   - Creates server + автоматически creator → owner role.
   - Создаёт default channels: `#общее` (text), `общая комната` (voice).
2. **`CreateServerModal`**:
   - Поля: имя (required, 2-64), иконка (drag-n-drop, optional).
   - При submit → POST /servers → switch to new server.
3. **`JoinServerModal`**:
   - Input для invite code.
   - При validation → preview server (как в T-013 onboarding).
   - При accept → POST /invites/:code/accept → переключение на новый сервер в шелле.
4. **`ServerRail` кнопка «+»**:
   - Click → small popover с двумя опциями: «создать сервер» или «принять инвайт».
   - Открывает соответствующую модалку.
5. **`InviteModal`** — генерация инвайт-кодов:
   - Доступ для admin/owner текущего сервера.
   - Список существующих активных инвайтов с метриками (uses / maxUses, expiresAt).
   - Кнопка «создать новый» → опции (expires, maxUses) → POST /invites → результат с copy-to-clipboard URL'а вида `https://kakdela.example.com/?invite=CODE`.
   - Кнопка revoke на каждом активном.
6. **Server settings** (в правой колонке) — minimal: rename server, change icon, view audit log link, кнопка «leave server» (для не-owner'ов) / «delete» (для owner'а).
7. **WelcomeScreen** (переехал из T-013, см. `designs/final-onboarding.jsx`) — post-auth экран с тремя карточками:
   - «зайти к друзьям» — открывает `JoinServerModal` (поле для кода/ссылки).
   - «завести свою комнату» — открывает `CreateServerModal` (с подсказкой про шаблоны на будущее).
   - «вернуться к своим» — список серверов user'а, клик переключает шелл.
   - Сверху: тонкая навбар-полоса (логотип «кд», self-hosted · версия, аватар user'а), приветствие «привет, {displayName}. как дела?», снизу — две info-карточки («что такое свой инстанс», «горячие клавиши»).
   - Показывается из `Router.tsx`, когда user authed и `localStorage['kd:last-channel']` пуст (первый вход / нет других серверов).
